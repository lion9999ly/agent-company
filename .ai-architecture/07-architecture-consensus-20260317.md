## 一、问题诊断

### 问题1：数据获取效率差距

**现象**：豆包超然模式/Gemini Deep Research 一次对话即可获取丰富数据，我们的工具链层层降级屡屡碰壁。

**根本原因**：

| 维度 | 豆包/Gemini | 我们现状 | 差距 |
|------|-------------|----------|------|
| 架构 | 服务端一体化 | 客户端工具链拼凑 | 本质差距 |
| 数据源 | 预集成API | 实时爬取 | 稳定性差距 |
| 网络 | 云端IP白名单 | 本地受限IP | 环境差距 |
| 合规 | 商业授权 | 裸HTTP请求 | 风险差距 |

### 问题2：Echo(CPO)角色冗余

**现象**：用户升级为Claude Code后，Echo(CPO)的意图理解、任务拆解、结果汇报功能已被覆盖。

**冗余判定**：**部分冗余**

- ❌ 已被覆盖：意图理解、任务拆解、工具调用、结果汇报
- ✅ 仍有价值：长期记忆、跨会话协调、行为审计、熔断保护

## 二、共识方案

### 方案A：数据获取架构升级

**目标**：达到与豆包/Gemini同等的数据获取效率

**路径**：

```
P0（立即）: 接入第三方搜索API
├── SerpAPI: Google搜索结果API
├── Exa.ai: AI原生搜索引擎
└── Tavily: AI Agent专用搜索API

P1（短期）: 云端Agent执行层
├── 将数据采集任务部署到云端
├── 使用云端IP绕过本地限制
└── 支持 headed/headless 模式切换

P2（中期）: 分布式爬虫框架
├── Scrapy + Redis 集群
├── 代理IP池轮换
└── 验证码自动处理
```

**投入产出**：

| 阶段 | 投入 | 预期效果 |
|------|------|----------|
| P0 | API订阅费（$50-200/月） | 搜索类任务效率提升90% |
| P1 | 云服务器费用（$20-50/月） | 网络限制问题解决 |
| P2 | 开发周期（2-4周） | 复杂采集任务效率提升 |

### 方案B：Echo(CPO)重新定位

**目标**：将Echo从"调度中枢"转型为"基础设施守护者"

**新职责**：

| 角色 | 职责 | 对Claude Code价值 |
|------|------|-------------------|
| 长期记忆管家 | 项目上下文、用户偏好、历史决策 | 补足Claude Code会话级记忆限制 |
| 跨会话协调器 | 任务状态持久化、依赖追踪、进度恢复 | 支持跨会话任务恢复 |
| 行为审计官 | Agent越权检测、熔断保护、红线拦截 | 补足Claude Code无审计能力 |
| 模型路由守门 | 异构模型路由、双模型评审降级 | 提供多模型选择 |

**架构调整**：

```python
# 原：Echo作为调度中枢
User → Echo(CPO) → CTO/CMO → Result → Echo → User

# 新：Echo作为基础设施
User ↔ Claude Code → CTO/CMO → Result
              ↓
         Echo(守护者)
         ├── memory/ (长期记忆)
         ├── .ai-state/ (状态持久化)
         └── hooks/ (行为审计)
```
