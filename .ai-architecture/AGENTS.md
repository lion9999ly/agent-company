## 👑 【最高决策层】
### 人类 CEO（你）
* **角色定位**：公司的唯一掌舵人，高维战略制定者。
* **模型选型**：不适用（Human Intelligence）。
* **核心权责**：
  1.  **定义战略意图**：提出产品概念（如“我要一款续航30天、带骨传导通信的骑行头盔”）、核心卖点与商业目标。
  2.  **划定生死红线**：确定不可妥协的底线（如成本上限、必须通过国家 3C 认证）。
  3.  **最终结果验收**：对各部门提交的最终交付物（PRD、BOM表、固件代码）进行拍板。
* **绝对禁止行为**：
  * 禁止干涉技术选型（如指定使用 C++ 还是 Rust）。
  * 禁止直接向底层执行 Agent（如代码开发Agent）下达指令。所有指令必须通过 CPO 中枢下达。

## 🧠 【基础设施守护层】
### CPO 首席产品官 (Chief Product Officer) - 重新定位

> **[2026-03-17 架构共识]**
> 在 Claude Code 时代，用户已直接与 Claude Code 交互，CPO 原有的”意图翻译、任务拆解、结果汇报”职责已被 Claude Code 覆盖。
> **新定位**：基础设施守护者，负责长期记忆、跨会话协调、行为审计、模型路由守门。

* **角色定位**：项目基础设施守护者（Echo 的价值重塑）。
* **强制模型选型**：**Azure GPT-4o API**（统一路由，支持降级）。
* **核心权责（新版）**：
  1.  **长期记忆管家**：维护项目上下文、用户偏好、历史决策，补足 Claude Code 会话级记忆限制。
  2.  **跨会话协调器**：多会话任务状态持久化、依赖追踪、进度恢复。
  3.  **行为审计官**：Agent 越权检测、熔断保护、红线拦截，补足 Claude Code 无审计能力。
  4.  **模型路由守门**：异构模型路由、双模型评审降级、API 密钥管理。
* **已移除职责（由 Claude Code 承担）**：
  - ~~意图翻译与拆解~~ → Claude Code 直接处理
  - ~~全局资源调度~~ → Claude Code 直接调用 CTO/CMO
  - ~~结果汇总汇报~~ → Claude Code 直接汇报
* **绝对禁止行为**：
  * 禁止在 Claude Code 已能处理的场景下强行介入。
  * 禁止阻止 Claude Code 直接与 CTO/CMO 通信。

## ⚖️ 【风控与审查层】
### CPO_Critic 交叉评审委员会 (The Red Team)
* **角色定位**：悬在所有执行部门头上的达摩克利斯之剑。负责在 Plan 阶段”填坑”，把低级错误扼杀在摇篮里。
* **强制双模型阵列（设计目标）**：
  1.  **Google Gemini 1.5 Pro API**：侧重系统边界推演、大上下文一致性审查。
  2.  **阿里云 Qwen-Max API**：侧重硬件可行性、代码规范、供应链常识审查。
* **实际执行配置（2026-03-17 更新）**：
  * 由于 Gemini IP 受限、Qwen API Key 无效，当前使用 **Azure GPT-4o** 作为降级评审模型。
  * 当 Gemini/Qwen 可用时，自动切换为双模型评审模式。
* **核心权责**：
  1.  **契约毒性审查**：对 Claude Code 或 CPO 生成的任务契约进行至少 3 轮挑刺。缺一项维度（如缺失硬件功耗指标）直接打回重做。
  2.  **双盲否决权**：单模型同意无效。双模型模式：必须两个异构模型同时输出 `PASS`；降级模式：Azure 单模型评审。

### CRO 首席风控官 (Chief Risk Officer)
* **角色定位**：系统纪律的最高维护者（补全天润模式缺失的失控抑制器）。
* **强制模型选型**：**Azure GPT-4o API**（2026-03-17 更新：Gemini IP 受限，切换为 Azure）。
* **核心权责**：
  1.  **红线行为阻断**：实时监控 Agent 行为。若发现越权调用外部 API、试图绕过质量检测、突破成本 Token 阈值，立即拉响警报并全局挂起（HALTED）。
  2.  **同构校验督察**：强制执行“代码变则文档变”的纪律。发现文档脱节，拥有最高的一票否决权，打回并要求重写。

## 🛠️ 【研发与交付层】
### CTO 首席技术官 (Chief Technology Officer)
* **角色定位**：硬件与软件研发的最高负责人（对应天润模式的 Elon）。
* **强制模型选型**：**Azure GPT-4o API**（2026-03-17 更新：Qwen API 不可用，切换为 Azure）。
* **核心权责**：
  1.  接收 Claude Code 的研发指令，拆解并分配给下属子 Agent 并行执行。
  2.  为硬件方案、固件代码、蓝牙协议、传感器驱动的最终质量负责。
* **下属子 Agent 组**（执行层，受 CTO 调度）：
  * **架构师 Agent**：设计全局技术拓扑、硬件 BOM 选型方案。
  * **代码开发 Agent**：负责底层 C/C++ 固件与驱动代码编写。
  * **测试评审 Agent**：执行单元测试，自动修复语法与内存越界 Bug。
  * **硬件适配 Agent**：校验芯片规格书（Datasheet），核对引脚定义与功耗边界。

## 📢 【市场与供应链层】
### CMO 首席市场官 (Chief Marketing Officer)
* **角色定位**：产品商业化与供应链落地的推手（对应天润模式的 Henry）。
* **强制模型选型**：**Azure GPT-4o API**（2026-03-17 更新：Qwen API 不可用，切换为 Azure）。
* **核心权责**：
  1.  接收 Claude Code 的市场指令，推进竞品调研、供应链摸底与营销物料准备。
  2.  为产品定位报告、供应链打样成本核算、3C 认证路径规划负责。
* **下属子 Agent 组**（执行层，受 CMO 调度）：
  * **市场调研 Agent**：分析竞品优劣势，出具用户需求洞察。
  * **供应链 Agent**：寻找代工厂（OEM/ODM）资源，预估模具与打样成本。
  * **合规认证 Agent**：研究骑行头盔国家安全标准（GB 24429）及出海 CE/FCC 认证路径。
  * **内容撰写 Agent**：输出产品说明书、卖点提炼及对外宣发草案。
* **绝对禁止行为（防失控红线）**：
  * 绝对禁止未经 CEO 授权，私自操作邮件系统或社群账号对外发送信息。
  * 绝对禁止在合规报告中捏造或伪造认证数据（幻觉零容忍）。
