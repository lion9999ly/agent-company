## 方案描述

本方案旨在生成一个用于向供应商和投资人展示的智能骑行座舱 HUD 体验 Demo。该 Demo 是一个零依赖的单 HTML 文件，通过键盘交互和自动剧本，直观呈现 V1 核心功能，并突出“被保护着”的核心差异化。

**1. 整体架构与视觉框架**
- [偏好·--] **视觉风格**：采用深色背景模拟头盔内部视野，HUD 元素使用高饱和度、高亮度的功能色（导航蓝、预警红、组队青、音乐紫、速度白），并辅以微弱的呼吸动画，以在普通显示器上清晰传达“信息浮于视野前方”的 HUD 沉浸感。回应约束：CTO-16。
- [判断·高] **布局骨架**：采用“右下角主信息区 + 顶部状态条 + 中央瞬时弹层”的三层布局。右下角主信息区承载导航、速度等核心巡航信息；顶部状态条显示连接状态、电量等次要信息；中央区域预留给最高优先级的 ADAS 预警和来电等瞬时强提醒。此布局收敛于安全导向，信息密度低，确保一瞥可读。回应约束：CDO-4, CMO-2, CMO-4。
- [判断·高] **状态机模型**：采用“基础巡航态 + 瞬时覆盖态”的两层模型，而非 7 个平级页面。基础态为“导航巡航”，其上可被更高优先级的“ADAS预警”、“来电”、“语音交互中”等状态覆盖。覆盖态消失后自动回退至基础态。这简化了优先级逻辑，确保安全预警能无条件抢占。回应约束：CTO-9, CTO-11。

**2. 核心功能演示与叙事节奏 (自动剧本)**
自动剧本设计为约 90 秒的循环演示，叙事节奏为：建立安全信任 → 展示便利性 → 呈现社交乐趣。
- [判断·低] **第一阶段：建立安全 (0-40秒)**。开机动画（系统自检+品牌LOGO）后，直接进入导航巡航态。随后，依次触发三个核心 ADAS 预警场景：1) **BSD盲区预警**（左侧出现红色闪烁图标与“注意侧后方”文字）；2) **开门杀预警**（中央大幅红色闪烁警示框与“左侧危险！”）；3) **前车碰撞预警 (FCW)**（中央红色闪烁车距条与“前车过近！”）。每个预警持续 3-4 秒，视觉冲击力强，红色独占。回应约束：CDO-3, CMO-1, CMO-3, CTO-6。
- [判断·中] **第二阶段：展示便利 (40-70秒)**。预警结束后，模拟语音交互：“导航到外滩”。右下角导航信息更新为蓝色箭头、距离和路口放大图。随后，模拟来电（顶部状态条显示），并通过语音指令“接听”完成接听，展示语音消灭掏手机的能力。回应约束：CDO-2, CMO-8。
- [判断·低] **第三阶段：呈现乐趣 (70-90秒)**。模拟 Mesh 组队通信激活，右下角信息区变为青色，显示“车队在线：3人”。同时，音乐播放状态在右下角以紫色图标显示。最后所有状态收起，回归纯净导航界面，循环重新开始。回应约束：CDO-8。

**3. 交互与实现关键点**
- [判断·中] **键盘控制全覆盖**：支持数字键（1:导航，2:BSD预警…）直接触发状态；空格键暂停/继续自动剧本；`A`键切换自动/手动模式；`R`键重置Demo；`Esc`键强制清除所有覆盖态。确保现场演示稳定、容错。回应约束：CTO-10, CTO-13。
