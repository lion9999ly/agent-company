# HUD Demo 视觉与交互设计方案

## 方案描述

本方案旨在生成一个零依赖、单HTML文件的HUD交互演示，核心目标是向供应商与投资人直观传达“智能骑行座舱”如何通过HUD、语音与ADAS联动，**消灭停车掏手机**的核心价值。方案严格遵循所有约束，并围绕一个3分钟的自动剧本展开。

**1. 视觉架构与布局 (回应CDO约束1, 2, 3)**
- [偏好·--] 采用 **固定视觉骨架**：屏幕右下角为**主信息区**（常驻速度、导航箭头与距离），顶部为**轻提示条**（显示连接状态、时间、电量），中央区域为**瞬时弹层**（专用于高优先级ADAS预警与系统通知）。此布局不做全局自适应重排，确保信息位置稳定，利于余光扫视。
- [判断·高] **信息密度极简**：任何时刻，主信息区元素不超过4个（如：速度值、导航箭头、下一个路口名、剩余距离）。字体采用高对比度、无衬线字体，主信息（如速度、预警文字）字号确保在模拟的2-3米虚像距离下清晰可辨 — 基于：头盔HUD的物理/认知限制与竞品分析。
- [判断·中] **全彩功能色语义系统**：
    - **速度/状态信息：白色** (中性、高亮)
    - **导航指引：蓝色** (遵循行业惯例)
    - **ADAS预警（碰撞、盲区）：红色** (最高优先级，结合动态)
    - **Mesh组队通信：青色** (与导航蓝区分，体现社交属性)
    - **音乐/媒体控制：紫色** (娱乐相关)
    — 基于：汽车HUD与专业UI设计规范类推，以颜色降低认知负荷。

**2. 状态机与页面态设计 (回应CTO约束2, 8, 9, 11)**
- [判断·高] 定义7个核心页面态，它们不是独立页面，而是基于同一HUD骨架的**状态层叠加**：
    1.  **待机/开机自检**：品牌Logo动画，系统状态检查。
    2.  **主骑行界面**：基础状态，显示速度、导航概览。
    3.  **详细导航引导**：触发导航后的状态，显示更详尽的转向信息。
    4.  **ADAS预警（中央弹层）**：高优先级覆盖态，如“左侧盲区有车！”。
    5.  **语音交互反馈**：显示语音指令的识别结果与执行状态。
    6.  **组队通信激活**：显示队内频道、成员状态。
    7.  **系统通知**：低优先级提示，如“蓝牙已连接”。
- [判断·高] 实现**优先级抢占机制**：例如，当骑手正在查看导航时（态3），若触发ADAS预警（态4），界面会立即强制切换到态4。预警结束后，根据逻辑自动返回前一个状态（如态3）或基础态（态2）。此逻辑通过一个轻量级状态机管理 — 基于：状态机设计原则与HUD多态叠加场景。

**3. 交互与自动剧本 (回应CTO约束3, 4, 12; CMO约束1, 3, 5)**
- [判断·中] **键盘事件映射**：使用数字键1-7直接切换7个基础状态，用于手动演示。同时定义功能键（如空格键模拟语音触发，`B`键模拟盲区预警，`N`键模拟新导航指令），用于触发更复杂的、带有时序的交互事件链，以验证状态机的抢占与恢复逻辑 — 基于：交互状态机测试经验类推。
- [判断·中] **自动剧本（3分钟叙事）**：采用声明式时间轴数组驱动。剧本围绕“**上车-骑行-遇险-组队-抵达**”主线：
    1.  **(0-30s) 开机与连接**：展示品牌与自检，建立可靠感。
    2.  **(30-60s) 语音设置导航**：模拟用户说“导航去西山咖啡馆”，HUD显示语音反馈并进入详细导航态，**核心展示免掏手机**。
    3.  **(60-100s) 平稳骑行**：展示基础HUD信息（速度、简易导航箭头）。
    4.  **(100-110s) 突发ADAS预警**：模拟“左侧盲区有车”红色预警（中央弹层剧烈闪烁），**核心展示安全差异化**。预警持续数秒后自动消失。
    5.  **(110-130s) 组队通信加入**：模拟收到组队邀请并加入，HUD提示色变为青色。
    6.  **(130-180s) 接近目的地**：导航信息细化，最终显示“即将到达”，完成叙事。
    — 基于：类似Demo/动画编排工程经验类推，确保节奏可控、价值点突出。

**4. 技术实现与代码结构 (回应CTO约束1, 5, 6, 10, 14)**
- [判断·高] 使用纯原生技术栈：HTML (结构)、CSS (样式与关键帧动画)、Vanilla JavaScript (逻辑)。所有资源（如字体通过base64嵌入）内联，确保单文件零依赖 — 基于：该约束下外部框架不可用，且原生方案最可控。
- [判断·中] 代码模块化组织：在单个HTML文件内，通过`<script>`标签分隔逻辑模块，大致顺序为：常量配置、DOM元素引用、状态机定义、渲染函数、输入监听器、自动剧本时间轴、调试面板逻辑。这保证了代码的可读性与可维护性 — 基于：前端单文件Demo可维护性经验类推。
- [偏好·--] 集成一个可折叠的**调试面板**，默认隐藏，按`D`键显示。面板实时显示当前主状态、覆盖状态、剧本步骤和所有按键映射，便于演示者现场解释和故障兜底 — 技术偏好，无对错。

**5. 风险视觉表达 (回应CDO约束7; CMO约束3)**
- [判断·低] **ADAS预警的视觉冲击力**：中央弹层使用高饱和度红色背景、白色粗体文字。结合CSS动画实现“呼吸式”明暗闪烁与轻微震动（模拟头盔提示），并配以警告图标。这是最高优先级的视觉表达，旨在瞬间抓住注意力 — 基于：安全第一原则与ADAS功能重要性个人推测。

## 自评：最大风险
**最大风险在于“视觉保真度与性能的平衡”**。为了在单文件内实现平滑的动画（如预警闪烁、状态过渡）和嵌入字体，CSS和Base64编码可能会使文件体积膨胀。同时，复杂的CSS动画与JavaScript状态机在低性能设备上的运行流畅度存在不确定性。这可能导致在投资人电脑上演示时出现卡顿，破坏沉浸感和专业感。**缓解策略**：严格控制动画复杂度，使用硬件加速的CSS属性（`transform`, `opacity`），并在剧本开始前进行简单的性能检测与提示。
