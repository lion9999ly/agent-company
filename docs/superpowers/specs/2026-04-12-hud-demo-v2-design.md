## 设计摘要

重做智能骑行座舱 HUD Demo HTML 文件，严格遵守 SKILL.md v2.0 的 14 条验收标准。核心变化：
- 从"四角布局"改为"两条竖向显示带 + 中央透明"的物理模型
- 新增三种光学模式，单绿模式必须使用亮度编码而非颜色
- ADAS 预警边缘闪烁在竖条内部边框上

## 1. 物理布局模型

### 布局结构

```
┌─────────────────────────────────────────────────┐
│ ┌──────┐                           ┌──────┐     │
│ │  LT  │                           │  RT  │     │
│ │ 速度 │                           │ 标志 │     │
│ ├──────┤                           ├──────┤     │
│ │ 导航 │     中央透明区域          │      │     │
│ │ 地图 │     （路面可见）          │      │     │
│ ├──────┤                           ├──────┤     │
│ │  LB  │                           │  RB  │     │
│ │ADAS  │                           │组队  │     │
│ └──────┘                           └──────┘     │
│          ←── 辐辏安全间距 ──→                    │
└─────────────────────────────────────────────────┘
```

### 硬约束

1. 信息只能出现在左右两条竖向显示带内
2. 中央区域必须完全透明——不放任何 HUD 信息
3. 中央间距满足辐辏约束（默认 ≥ 人因自然约束，可调）
4. 显示带位于面镜高度中偏上位置
5. 显示区域大小/位置可通过 sandbox 调节（验收标准 #14）

### 区域分配

| Zone | 位置 | 主要内容 | 次要内容 |
|------|------|---------|---------|
| LT | 左上 | 速度（km/h）+ 速度等级色 | 巡航/警告状态标签 |
| RT | 右上 | 交通标志（持续显示到下一个）| 状态信息（电量/信号/温度）|
| LB | 左下 | ADAS 预警方向 + LDW 虚线 | 来电接听/拒绝 |
| RB | 右下 | ADAS 预警详情 | Mesh 组队信息 |
| 左中段 | 左侧竖条中间 | 导航地图（可调显示级别）| — |

## 2. 三种光学模式

### 模式 A：OLED + FreeForm 棱镜

- FOV: 38°（可调）
- 色彩: 全彩 RGB
- 亮度: 高（到眼 ~10,000 nits 单绿 / ~1,500 nits 全彩）
- 信息编码: 颜色 + 亮度 + 大小 + 位置 + 闪烁
- 背景: 半透明叠加路面（反射率 ~50%，可调）

### 模式 B：全彩光波导

- FOV: 28-35°（可调）
- 色彩: 全彩 RGB
- 亮度: 低（到眼 <1,000 nits）
- 透过率: 70-85%（背景更清晰）
- 信息编码: 同模式 A

### 模式 C：单绿光波导

- FOV: 28-35°（可调）
- 色彩: **唯一色相 #00FF00（520nm 绿光）**
- 亮度: 中（到眼 ~1,500 nits）
- **严格规则**：
  - 不允许任何其他颜色，包括 ⚠️ 图标在内的所有元素都必须是 #00FF00
  - 信息区分只使用四种手段：亮度百分比、字号大小、位置、闪烁频率
  - 默认亮度分级：紧急100% + 闪烁、主要80%、次要50%、背景20%

### 验收标准 #1-2

- 同一个 ADAS 预警在三种模式下的编码方式必须有可见差异
- 单绿模式下不得出现 #00FF00 以外的任何色相

## 3. ADAS 预警规范

### 预警类型

FCW、BSD、Dooring、行人检测、LDW、交通标志识别

### 每个预警必须显示

- 威胁类型名称
- 方向（前/左/右）
- 距离（米）
- 关闭速度（km/h）
- TTC（Time To Collision，秒，倒计时显示）
- 威胁等级：提示（1Hz）/ 警告（2Hz）/ 紧急（4Hz）

### 方向映射（修正版）

| 方向 | 闪烁位置 | 说明 |
|------|----------|------|
| 前方威胁 (FCW/行人) | LT + RT zone 同时闪烁 | 双侧 zone 边框脉冲 |
| 左后方来车 (BSD) | 左侧竖条**左边缘（外边缘）**闪烁 | 竖条内部最左边 4px 亮条 |
| 右侧威胁 (Dooring) | 右侧竖条**右边缘（外边缘）**闪烁 | 竖条内部最右边 4px 亮条 |
| LDW 左侧压线 | 左侧竖条**右边缘（内边缘）**虚线闪烁 | 竖条内部靠中央的虚线 |
| LDW 右侧压线 | 右侧竖条**左边缘（内边缘）**虚线闪烁 | 竖条内部靠中央的虚线 |

**关键定义**：
- 外边缘 = 竖条远离中央的那条边（左侧条的左边、右侧条的右边）
- 内边缘 = 竖条靠近中央的那条边（左侧条的右边、右侧条的左边）
- 所有边缘闪烁都在竖条**内部**（宽度 4px），不超出显示带范围

### 验收标准 #4-7

- #4: ADAS 预警必须同时显示距离 + 关闭速度 + TTC
- #5: 多重预警并发时 TTC 最短的最大化，其他弱化但可见
- #6: BSD 左后方闪烁出现在左侧竖条内部左边缘（外边缘）
- #7: LDW 虚线闪烁出现在对应侧竖条内部靠中央的边缘（内边缘）

## 4. 状态机

### 7 种模式

`cruise / nav / call / music / mesh / warn / dvr`

### 优先级（高→低）

`warn > call > nav > mesh > music > dvr > cruise`

### 优先级抢占行为

- 高优先级触发时，低优先级信息**缩小到角落但不消失**（验收标准 #8）
- 预警结束后**自动恢复**之前状态，无需用户操作（验收标准 #9）

### 速度分级

| 等级 | 速度范围 | 显示策略 |
|------|---------|---------|
| S0 | 0-30 km/h | 全部信息显示 |
| S1 | 31-60 km/h | 全部信息显示 |
| S2 | 61-100 km/h | 导航降级为简易地图，隐藏音乐/组队详情 |
| S3 | >100 km/h | 只保留速度 + ADAS + 简化导航箭头 |

---

## 5. 导航显示

### 三级显示

1. **箭头模式**：转向箭头 + 距离
2. **简易地图**：路口简图 + 转向指示
3. **全信息地图**：完整路线地图

### 切换规则

- S2/S3 速度时自动降级为箭头或简易地图
- 用户可手动切换（sandbox 按钮）
- 导航地图显示在左侧竖条中段

## 6. 剧本（4 个）

### 1. commute（通勤）50s

日常通勤场景：启动 → 导航 → 来电抢占 → 自动恢复导航 → 到达

**语音交互**：导航切换 ×2

### 2. emergency（紧急）45s

密集 ADAS 预警场景：FCW → BSD → Dooring → 行人检测

**验收**：多重预警并发 + TTC 倒计时

### 3. group（组队）55s

摩友组队骑行：Mesh 组队 → DVR 录像 → 语音标记精彩瞬间 → 导航

**语音交互**：标记精彩瞬间 ×1

### 4. touring（摩旅）60s

长距离巡航场景：风景巡航 → DVR 生活记录 → 语音标记 → 交通标志识别 → 低速停靠拍照

**验收标准 #10**：语音标记精彩瞬间
**验收标准 #13**：交通标志持续显示在 RT zone 直到下一个标志

---

## 7. 语音交互

- 语音内容在**非可视区域**（画布底部或顶部）以字幕形式呈现（验收标准 #11）
- 字幕内容：用户语音 → AI 识别 → 执行结果
- 每个剧本至少 2 次语音交互演示

## 8. Sandbox 控件

必须提供以下可调参数：

| 控件 | 说明 |
|------|------|
| 光学模式切换 | F1/F2/F3 或按钮 |
| FOV 大小滑块 | 28°-50° |
| 显示区域位置/大小 | 宽度、高度偏移 |
| 中央间距调节 | 辐辏约束参数 |
| 背景透明度 | 0-100% |
| 背景图片上传 | 替换默认路面背景 |
| 单绿亮度分级 | 紧急/主要/次要/背景百分比 |
| 导航显示级别 | 箭头/简易地图/全地图 |
| 速度设置 | 0-300 km/h |

| 剧本播放控制 | 4个剧本 + 播放/暂停/停止 |

## 9. 技术约束

- 单文件 HTML（含 CSS + JS）
- DOM ID 命名：`zone-lt, zone-rt, zone-lb, zone-rb, bottom-bar, sandbox, timeline, boot-overlay, optics-indicator, hud-viewport-left, hud-viewport-right, speed-value`
- CSS 变量使用 `clamp()` 响应式
- 所有可调参数存储在全局配置对象，sandbox 修改后实时生效
- window 上暴露：`setMode, getMode, setSpeed, setOptics, emitWarning, emitEvent, playScenario`

## 10. 14 条验收标准（完整清单）

1. 三种光学模式切换后，同一个 ADAS 预警的编码方式必须可见差异（全彩用颜色，单绿用亮度）
2. 单绿模式下不得出现 #00FF00 以外的任何色相（包括所有图标和元素）
3. 中央区域必须完全透明，可见路面背景
4. ADAS 预警必须同时显示距离 + 关闭速度 + TTC
5. 多重预警并发时 TTC 最短的最大化，其他弱化但可见
6. BSD 左后方闪烁出现在左侧竖条内部左边缘（外边缘）
7. LDW 虚线闪烁出现在对应侧竖条内部靠中央的边缘（内边缘）
8. 优先级抢占后被抢占内容缩小但不消失
9. 预警结束后自动恢复之前状态
10. touring 剧本包含语音标记精彩瞬间
11. 语音交互以字幕形式出现在非可视区域
12. 路面背景透明度可调
13. 交通标志识别持续显示在 RT zone 直到下一个标志
14. 显示区域大小和位置可通过 sandbox 调节

## 11. 设计决策记录

### 边缘闪烁位置（用户反馈）

> 用户反馈："边缘要调整到竖条的内部，不可能超出竖条的范围。"

**决策**：所有边缘闪烁在竖条边框内部，宽度 4px，不超出显示带范围。
- 外边缘 = 竖条远离中央的边（内部最外侧 4px）
- 内边缘 = 竖条靠近中央的边（内部靠中央的虚线）

### 单绿模式图标色相（用户反馈）

> 用户反馈："单绿光波导，⚠️这个icon，都要只能是绿色。"

**决策**：单绿模式下所有视觉元素（包括 ⚠️ 图标）都必须是 #00FF00 色相，不得使用任何其他颜色。
