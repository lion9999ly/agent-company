# 产品锚点文档

> **VERSION: 20260412.1**

## 一句话定义

智能骑行座舱——将车规级 ADAS 安全感知能力首次带给摩托车骑手的全脸头盔

---

## 核心使命

消灭"停车掏手机"这个动作。从上车到下车，骑手全程不需要碰手机。

---

## 三、硬约束（不可逾越）

| 约束类型 | 具体要求 | 原因 |
|----------|----------|------|
| **形态约束** | 全盔形态，全内置 | 外挂设备破坏美感+风阻+风噪 |
| **外观约束** | 不能比普通全盔差 | 工业设计权重 ≥ 功能数量 |
| **续航约束** | ≥ 8 小时（重度使用） | 玩乐/摩旅用户骑行时长 |
| **重量约束** | ≤ 1.8kg（含全部电子） | 普通全盔 1.4-1.6kg，上限增加 200-400g |
| **安全认证** | ECE 22.06 / GB 811 | 头盔本体必须通过标准认证 |

## 六、端侧 AI 四层架构

### 架构总览

```
┌─────────────────────────────────────────────────────────────┐
│                    L3: 云端大模型                             │
│    复杂语义理解、多轮对话、好奇查询、OTA升级模型               │
└─────────────────────────────────────────────────────────────┘
                           ↕ 云端通信
┌─────────────────────────────────────────────────────────────┐
│                    L2: 端侧轻量模型                            │
│    指令纠错、意图分类、简单问答、离线基础能力                  │
│    模型: <100MB, 算力需求 <0.5 TOPS                          │
└─────────────────────────────────────────────────────────────┘
                           ↕
┌─────────────────────────────────────────────────────────────┐
│                    L1: DSP/专用芯片                            │
│    语音唤醒、降噪ENC、波束成形、音频编解码                    │
│    算力: DSP处理, 延迟 <100ms                                │
└─────────────────────────────────────────────────────────────┘
                           ↕
┌─────────────────────────────────────────────────────────────┐
│                    L0: 硬件传感器                              │
│    麦克风阵列、IMU、摄像头、环境光传感器                      │
└─────────────────────────────────────────────────────────────┘
```

### 各层详解

#### L0: 硬件传感器层

| 传感器 | 规格 | 用途 |
|--------|------|------|
| **麦克风阵列** | 双麦 ENC + 防风套 | 语音拾取 + 降噪 |
| **IMU (6轴)** | 加速度+陀螺仪 | ADAS 倾角补偿、姿态检测 |
| **摄像头** | 前向 1080p + 后向可选 | 行车记录 + ADAS 感知 |
| **环境光传感器** | ALS | HUD 亮度自适应 |
| **GPS** | 高精度定位 | 导航 + ADAS 场景判断 |

#### L1: DSP/专用芯片层

| 功能模块 | 实现方式 | 性能目标 |
|----------|----------|----------|
| **语音唤醒** | DSP 本地关键词检测 | 延迟 <200ms, 误唤醒 <1次/天 |
| **ENC降噪** | 双麦波束成形 + RNNoise | 80km/h 以下可用 |
| **物理防风** | 下巴密封 + 麦克风防风套 | 风噪降低 ~80% (60km/h以下) |
| **音频编解码** | DSP 硬件加速 | 低功耗，实时处理 |

**关键决策**: V1 不上 ANC（主动降噪），采用物理隔音 + ENC + 云端增强方案

#### L2: 端侧轻量模型层

| 功能 | 模型规格 | 算力需求 |
|------|----------|----------|
| **指令纠错** | <50MB 模型 | <0.2 TOPS |
| **意图分类** | 规则+轻量ML | <0.1 TOPS |
| **离线指令集** | 硬编码规则 | 无算力需求 |
| **简单问答** | 本地知识库检索 | <0.3 TOPS |

**离线指令集（V1 支持）**:
- 导航开始/暂停/继续
- 接听/拒绝来电
- 播放/暂停音乐
- 开始/停止录制
- Mesh 组队/离队
- 呼叫紧急联系人

#### L3: 云端大模型层

| 功能 | 实现方式 | 调用时机 |
|------|----------|----------|
| **复杂语义理解** | GPT-4o/o3 | 多轮对话、模糊指令 |
| **ASR 语音识别** | 云端 ASR API | L1 唤醒后上传音频 |
| **TTS 语音合成** | 云端 TTS API | 回复播报 |
| **好奇查询** | 云端知识库 + 搜索 | V2 功能 |
| **OTA 模型升级** | 云端推送 | 模型迭代 |

### 端云协同策略

| 场景 | 处理层级 | 原因 |
|------|----------|------|
| **唤醒词检测** | L1 (DSP) | 必须本地，低延迟 |
| **基础指令执行** | L2 (端侧) | 离线可用，隐私保护 |
| **复杂语音交互** | L3 (云端) | 需大模型理解能力 |
| **极限噪音场景** | L3 (云端大模型纠错) | 90dB+ 场景准确率提升 40% |
| **无网络隧道** | L2降级 | 离线指令集兜底 |

## 七、ADAS 差异化定位

### 核心定位

**"首个将车规级 ADAS 带给摩托车骑手的全脸头盔"**

### 与汽车 ADAS 的差异

| 维度 | 汽车 ADAS | 摩托车 ADAS (我们) |
|------|-----------|-------------------|
| **载体** | 车身固定 | 头盔随骑手移动 |
| **倾角补偿** | 不需要 | 必须（压弯 30-45°+） |
| **感知范围** | 全车周边 | 前向为主 + BSD |
| **误报容忍** | 中等容忍 | 极低容忍（干扰骑手注意力=危险） |
| **核心场景** | 高速公路、城市道路 | 摩托车专属场景（穿行、小障碍物） |

### V1 ADAS 功能范围

| 功能 | 状态 | 优先级 | 备注 |
|------|------|--------|------|
| **BSD (盲区检测)** | V1 必须 | P2 | 复用 Luna6，适配摩托车视角 |
| **开门杀预警** | V1 必须 | P2 | 路边停车场景高危 |
| **FCW (前碰撞预警)** | V1 必须 | P2 | 需专项优化小障碍物识别 |
| **行人检测** | V1 必须 | P2 | 城市骑行高危场景 |
| **LDW (车道偏离)** | V2 | -- | 高误报风险，V1 不做 |

### 小障碍物识别专项优化

摩托车场景高危小障碍物：
- 坑洼、路面杂物
- 近距离电动车、自行车
- 行人（尤其是横穿）
- 单车灯（易误判为路灯）

**技术策略**: 专项数据采集 + 算法适配，优先级高于 LDW

### ADAS 算法修改量估算

| 功能 | 汽车迁移修改量 | 原因 |
|------|----------------|------|
| FCW | 40-60% | 需增加小障碍物识别优化 |
| BSD | 20-40% | 对车道中心线依赖较低 |
| LDW | 60-80% | 高误报率，摩托车压弯场景冲突严重 |

### 研发周期

| 版本 | 功能范围 | 周期 |
|------|----------|------|
| V1 (FCW+BSD+行人) | 核心安全功能 | 8-12 月 |
| V2 (+LDW) | 全功能 ADAS | 额外 4-6 月 |
