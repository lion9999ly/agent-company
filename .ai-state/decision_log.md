## [2026-04-06] 光学路径选择：OLED + FreeForm 棱镜

### 背景
- V1 HUD 需确定光学显示方案
- 两条路径：A（OLED+FreeForm）、B（MicroLED+光波导）

### 选项
- A: OLED + FreeForm 棱镜（单色/全彩可选）
- B: MicroLED + 衍射光波导（单绿）
- C: 等待 MicroLED 全彩成熟（18-36 月）

### 决策
- 选择：A（OLED + FreeForm）
- 原因：
  1. **量产周期**：A 方案 5-7 月，B 方案 18-36 月
  2. **供应链成熟度**：OLED 已量产，光波导仍在验证
  3. **成本可控**：OLED 单价 300-500 元，光波导 800-1500 元
  4. **V1 目标**：快速验证市场，不追求极致技术

### 影响
- 技术：放弃光波导透明感，但获得量产确定性
- 时间：缩短 12-24 月上市周期
- 成本：降低 50% 光学成本

### 后续行动
- 联系歌尔/至格确认 FreeForm 棱镜供应
- 验证 OLED + FreeForm 在头盔震动环境表现

## [2026-04-12] smolagents 替代 deep_research 搜索层

### 背景
- 当前搜索层硬编码在 pipeline.py
- 需支持可插拔搜索工具（Tavily/Doubao）

### 选项
- A: 继续维护自建搜索层
- B: 整合 smolagents Tool 框架
- C: 整合 LangChain open_deep_research

### 决策
- 选择：B（整合 smolagents）
- 原因：
  1. **Tool 可插拔**：继承 Tool 类，30-50 行实现新工具
  2. **LiteLLM 原生对接**：LiteLLMModel 直接使用
  3. **step_callbacks**：可接入 Critic/KB hook
  4. **Windows 兼容**：v1.13+ 已修复主要问题

### 为什么不选 C（open_deep_research）
- 搜索层硬编码，SearchAPI Enum 不可扩展
- 需修改核心代码适配 doubao 中文搜索
- 长期 Merge 成本 > 自建维护

### 影响
- 技术：废弃自建搜索层逻辑
- 时间：缩短 2 周开发
- 成本：减少代码量 ~500 行

### 后续行动
- 实现 DoubaoSearchTool 和 TavilySearchTool
- 端到端测试验证
- 研究 step_callbacks 正确注册方式
