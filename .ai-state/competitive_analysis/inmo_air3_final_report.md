## 一、数据采集方法论总结

### 成功的采集策略

| 数据源 | 成功率 | 适用场景 | 技术方案 |
|--------|--------|----------|----------|
| 官网HTML | ✅ 100% | 产品规格、价格 | requests + BeautifulSoup |
| 海外官网 | ✅ 80% | 竞品规格 | requests (无反爬) |
| 电商平台 | ❌ 0% | 价格、评价 | 需登录/被反爬拦截 |
| WebSearch | ❌ 0% | 综合搜索 | API限制返回空 |
| WebFetch | ⚠️ 30% | 单页面抓取 | 部分网站支持 |
| 小红书页面 | ⚠️ 50% | 用户评价 | 动态渲染，需JS执行 |

### 失败原因分析

| 失败项 | 根因 | 替代方案 |
|--------|------|----------|
| 京东/天猫 | 强反爬+登录墙 | 手动采集/用户浏览器 |
| B站API | 412反爬响应 | 视频页面手动浏览 |
| WebSearch | API Key未配置 | 配置搜索API |
| 知乎/Reddit | 403禁止爬取 | 无公开API |

### 最佳实践

1. **优先访问官网** - 反爬弱，数据权威
2. **海外站点更友好** - 全球化产品官网通常无反爬
3. **避免电商平台** - 反爬最严格
4. **动态页面需JS** - 使用Selenium/DrissionPage

## 六、经验教训总结

### 技术层面

1. **反爬应对策略分级**
   - Level 1 (低): 官网 → requests直接获取
   - Level 2 (中): 动态渲染 → Selenium/DrissionPage
   - Level 3 (高): 登录墙 → 用户浏览器复用
   - Level 4 (极高): 电商 → 手动采集

2. **数据源优先级**
   - 官方网站 > 科技媒体 > 电商平台 > 社交平台

3. **工具选择**
   - requests: 静态页面，效率最高
   - DrissionPage: 动态页面+反爬绕过
   - browser-use: 复杂交互（但Windows兼容问题）

### 流程层面

1. **先验证数据源可访问性**
2. **优先获取官方权威数据**
3. **记录每个源的失败原因**
4. **建立采集成功率矩阵**
