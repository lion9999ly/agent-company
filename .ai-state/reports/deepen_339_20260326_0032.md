# Horizon Continental 长稳测试方案补充研究报告  
## 主题：拍屏长稳（行车+泊车）的完整技术参数、竞品对比、适配度评估

---

## 一、执行摘要

本次研究的核心，不是讨论“拍屏长稳要不要做”，而是回答一个更本质的问题：**对于智能骑行头盔 V1，什么样的长稳验证体系，既能真实反映用户体验，又能支撑歌尔 JDM 做工程闭环和量产验收。**

基于 CTO、CMO、CDO 三方分析，我的最终判断非常明确：

**拍屏长稳必须做，但它解决的是“用户可见层稳定性”问题，不是完整验收问题。V1 不能采用纯拍屏方案，必须采用“拍屏 + 系统日志 + 事件触发回录”的三层组合方案。**

原因很简单：

1. **第一性原理看本质**：用户买的不是“测试方法”，而是“长途骑行时 HUD 不黑、不闪、不乱、录像不断、语音不崩”的可靠体验。拍屏能证明“看到了什么”，日志能解释“为什么发生”，事件回录能把问题闭环。缺一不可。
2. **奥卡姆剃刀做取舍**：V1 不需要上完整车规级重型验证平台，也不需要为了“像汽车项目”而堆复杂系统。最短路径就是：外部拍屏记录用户视角，内部日志记录系统状态，异常时自动保留前后 30–60 秒证据。
3. **苏格拉底追问纠偏**：如果只做拍屏，团队其实是在解决一个 XY 问题——看起来像在验证稳定性，实际上只能看到表象，无法定位根因，也无法支撑量产验收。反过来，如果只做日志，也是在解决另一个 XY 问题——能看到系统指标，却不知道用户是否真的看见闪屏、漏显、错位。

因此，本报告的裁决是：

- **V1 标准方案：方案 C——拍屏 + 日志 + 事件触发回录**
- **拍屏长稳适配度：高，建议纳入 V1 质量体系，评分 8/10**
- **定位：B 端质量能力，不作为 C 端主卖点；对外转译为“长途稳定、暴雨稳定、烈日可视、连续录像不死机”**
- **测试对象：HUD 显示链路、4K 录像预览/录制状态、ADAS 告警 UI、语音交互反馈 UI、静置待机/哨兵模式/充电/热浸/反复唤醒**
- **预算建议：单套测试工装 ￥8,000–20,000；软件投入约 4–6 人月**
- **验收原则：拍屏是必要条件，不是充分条件**

需要特别指出：三位角色整体没有偏离“智能骑行头盔”这一产品定义，没有出现改品类错误；但 CMO 和 CDO 在部分论述中引入了较多市场与设计延展内容，虽然有价值，但必须收束回本次研究主题——**长稳测试方案补充**。本报告已完成收束与裁决。

## 二、核心分析

### 1. 先定义问题：拍屏长稳到底在验证什么

“拍屏长稳（行车+泊车）”本质上不是拍一块屏，而是通过外部相机持续记录用户可见界面，验证长时间运行下是否出现以下问题：

- 黑屏
- 花屏
- 卡死
- 闪烁
- UI 错位
- 预警漏显
- 录像中断
- 导航异常

在汽车里，这通常对应 HUD、中控或调试屏；在本项目中，必须映射到智能骑行头盔的真实对象：

- **HUD 显示链路**
- **4K 录像预览/录制状态**
- **ADAS 告警 UI**
- **语音交互反馈 UI**
- **静止驻车/怠速/摘盔待机/哨兵模式**

因此，头盔版长稳测试必须拆成两类：

#### 行车长稳
覆盖真实骑行中的高负载并发场景，包括：
- 60–100km/h 风噪、振动、强光
- HUD 导航 + 蓝牙音乐 + 1080p 录像 + 语音唤醒并发
- 城市 / 高架 / 隧道 / 夜路 / 暴雨模拟

#### 泊车长稳
覆盖静置和反复唤醒场景，包括：
- 静置待机
- 哨兵模式
- 充电
- 热浸
- 反复开关机
- 摘戴检测

这一步非常关键。因为如果团队把“拍屏长稳”理解成“拍 HUD 有没有亮”，那就是典型 XY 问题。真正要验证的是：**头盔座舱在长时间、复杂工况下，是否仍然可信。**

---

### 2. 三种方案对比：为什么纯拍屏不够，纯日志也不够

#### 方案 A：纯拍屏

**配置**
- 工业相机 / 运动相机：1080p60 或 4K30
- 固定支架 + 偏振滤镜
- 时间戳同步

**典型成本**
- GoPro / Insta360 级：￥2,500–4,000 / 套
- 工业相机方案：￥5,000–12,000 / 套

**优点**
- 直观，能直接看到用户视角问题
- 对 HUD 可读性、闪烁、UI 异常特别有效
- 部署快，适合歌尔样机阶段

**缺点**
- 只能看到结果，看不到根因
- 无法识别 CPU 过载、内存泄漏、传感器异常
- 人工回看成本高，8 小时测试视频 = 8 小时审片

**结论**
适合做 V1 必上能力，但**不能单独承担验收主方案**。

---

#### 方案 B：系统日志长稳

**配置**
- SoC 性能日志：CPU / GPU / NPU 占用、温度、内存、水位
- 关键进程心跳
- 摄像头 / HUD / 语音 / 蓝牙链路状态
- 崩溃 dump、重启原因、掉帧统计

**典型成本**
- BOM 几乎不增加
- 软件投入约 2–4 人月搭建基础框架

**优点**
- 能定位根因
- 可自动统计 MTBF、异常率、热失控点
- 对双芯片 / 三芯片架构特别关键

**缺点**
- 对“用户是否看得清、是否闪”不敏感
- 前期埋点不规范，后期数据不可用

**结论**
这是量产必备主链路，但**不能替代用户视角验证**。

---

#### 方案 C：拍屏 + 日志 + 事件触发回录

**配置**
- 外部拍屏 1080p60
- 内部日志 1Hz–10Hz 采样
- 异常触发前后 30–60 秒视频 / 日志保留
- GPS / IMU / 车速 / 亮度同步

**典型成本**
- 单套测试工装约 ￥8,000–20,000
- 软件投入约 4–6 人月

**优点**
- 既能看见问题，也能定位问题
- 适合 HUD、ADAS、录像、语音并发场景
- 可形成量产前标准验收体系

**缺点**
- 集成复杂度高于纯拍屏
- 时间同步精度需控制在 **<100ms**

**结论**
这是 V1 的最优解，也是本报告的最终推荐方案。

---

### 3. 完整技术参数建议

以下参数必须保留，作为 Horizon Continental 长稳测试方案补充的基础版本。

#### 3.1 行车长稳参数

- **测试时长**：单次 **8h**，累计 **>200h**
- **场景覆盖**：城市 / 高架 / 隧道 / 夜路 / 暴雨模拟
- **速度覆盖**：**0–120km/h**
- **振动频段**：主频 **20–200Hz**
- **环境温度**：**-10℃ / 25℃ / 45℃ / 60℃ 热箱**
- **功能并发**：HUD 导航 + 蓝牙音乐 + **1080p 录像** + 语音唤醒

**验收指标**
- 黑屏 / 死机：**0 次 / 8h**
- HUD 关键信息漏显：**0 次**
- 录像中断：**0 次**
- 语音误唤醒率：**<2 次 / h**
- 系统重启：**0 次**
- 表面贴脸区温度：**<40℃**

这里要特别说明：CTO 在测试并发中写的是 **1080p 录像**，而 CMO / CDO 在产品能力表述中多次提到 **4K 摄像 / 4K 黑匣子录像**。这不是矛盾，而是测试分层问题。我的裁决是：

- **V1 产品能力可以定义为 4K 录像**
- **长稳主验收并发链路先以 1080p 作为稳定性基线**
- **4K 连续录像作为专项压力测试，不作为所有场景默认并发基线**

理由：V1 的核心不是把最重负载跑满，而是先证明核心链路稳定。否则容易把“验证产品可卖”误做成“验证极限性能”。

---

#### 3.2 泊车 / 静置长稳参数

- **测试时长**：**24h 静置 + 7 天待机循环**
- **场景**：哨兵模式、充电、反复开关机、摘戴检测

**验收指标**
- 待机异常唤醒：**<1 次 / 24h**
- 哨兵误触发率：**<5%**
- 低电保护触发准确率：**100%**
- 冷启动：**<5s**

这部分对头盔尤其重要，因为“泊车”在头盔语境下不是汽车停车，而是**静置待机、摘盔、充电、热浸、反复唤醒**。CTO 的映射是正确的，应纳入正式方案定义。

### 4. 竞品与替代方案对比

#### 4.1 测试工装竞品 / 替代方案

| 方案 | 规格 | 价格区间 | 优势 | 风险 |
|---|---:|---:|---|---|
| GoPro Hero 级拍屏 | 4K30 / 1080p60 | ￥2,500–4,000 | 快速部署 | 滚动快门、频闪误判 |
| Insta360 Ace / Action 级 | 4K30 / 60 | ￥2,000–3,500 | 防抖好、安装灵活 | 长时供电与散热一般 |
| 工业相机 | 1080p60 / 120，全局快门可选 | ￥5,000–12,000 | 频闪控制好、可同步 | 成本高、部署复杂 |
| 纯日志方案 | 1–10Hz 系统采样 | 软件投入为主 | 根因定位强 | 看不到真实显示体验 |
| 推荐组合方案 | 拍屏 + 日志 + 回录 | ￥8,000–20,000 / 套 | 最完整 | 集成复杂 |

#### 4.2 产品竞品启示

- **Eyelights / EyeRide**：问题集中在白天看不清、续航差、结构脆、长时间骑行后闪屏、安装复杂，说明需求成立，但长稳能力不足。
- **Sena**：通讯稳定、音频成熟，但更像外挂通讯件，不是一体化智能头盔座舱。
- **Shoei / Forcite**：要么偏传统盔加一点智能，要么偏科技感但整机完成度不足。

结论不是“竞品都不行所以我们也别做”，而是：**市场缺的不是功能概念，而是工程成熟度。**

### 5. 适配度评估

综合三方观点，我给“拍屏长稳（行车+泊车）”对本项目的适配度评分为：

## **8/10，高度建议纳入 V1 质量体系**

原因如下：

1. **高度适配 HUD/HMI 验证**
   - 能直接发现闪屏、黑屏、错位、漏显、延迟漂移等用户最敏感问题。
2. **适合样机阶段快速部署**
   - GoPro / Insta360 / 工业相机都可快速搭建。
3. **能支撑市场语言转译**
   - 对外可转译为“长途稳定、暴雨稳定、烈日可视、连续录像不死机”。
4. **但不能独立完成量产验收**
   - 近眼 HUD 拍摄失真、问题不可定位、数据量爆炸，决定了它只能是证据层，不是唯一主链路。

## 三、冲突与裁决

### 冲突点 1：V1 是否以 4K 连续录像作为长稳默认基线

- **CMO / CDO 倾向**：4K 摄像是产品价值的一部分，应强调。
- **CTO 倾向**：长稳并发链路建议以 1080p 录像作为基线。

**裁决**：  
**V1 产品定义保留 4K 录像，但长稳主基线采用 1080p 并发，4K 作为专项压力测试。**

**理由**：  
这是典型的“产品能力”和“验收基线”不能混为一谈。若一开始把所有场景都按 4K 并发验收，容易把 V1 拖入高功耗、高热、高不确定性的复杂度陷阱，偏离“先把高频核心场景做稳”的原则。

---

### 冲突点 2：拍屏长稳是质量能力还是传播卖点

- **CMO**：不是 C 端主卖点，是 B 端质量能力。
- **CTO / CDO**：更强调其作为体验验证抓手的工程价值。

**裁决**：  
**内部按质量能力建设，外部不讲“拍屏长稳”，只讲稳定结果。**

**理由**：  
用户不为测试方法买单，只为稳定体验买单。对外传播“拍屏长稳”没有意义，甚至会显得工程语言过重；但内部必须把它作为 V1 质量门槛。

---

### 冲突点 3：测试重点是“屏幕稳定”还是“整机可信”

- **CTO**：偏显示链路、日志链路、事件闭环。
- **CDO**：强调重量、热、佩戴疲劳、振动下可读性。
- **CMO**：强调用户最终感知的“安心”。

**裁决**：  
**测试对象从“屏”升级为“头盔座舱”，但执行上仍以 HUD/录像/语音/ADAS 四条链路为主。**

**理由**：  
如果只测屏，范围太窄；如果把所有佩戴体验都塞进拍屏长稳，又会失焦。正确做法是：拍屏长稳聚焦可见层稳定性，同时同步记录温度、功耗、存储、振动、主观佩戴反馈，形成“座舱可信度”判断。

---

## 四、明确结论与行动建议

### 1. 明确结论

1. **拍屏长稳必须做，但不能单独作为 V1 验收主方案。**
2. **V1 标准方案确定为：拍屏 + 系统日志 + 事件触发回录。**
3. **测试范围必须覆盖行车长稳与泊车长稳两大类。**
4. **拍屏长稳适配度高，评分 8/10，应纳入 V1 质量体系。**
5. **对外不宣传测试方法，对内作为量产门槛能力建设。**

---

### 2. 行动建议

#### P0：两周内确定测试架构
- 确认采用 **方案 C**
- 明确时间同步要求 **<100ms**
- 确认日志采样频率 **1Hz–10Hz**
- 定义异常触发前后 **30–60 秒**回录策略

#### P1：四周内搭建首套工装
优先组合：
- GoPro Hero 级或 Insta360 Ace / Action 级做快速验证
- 工业相机方案作为后续标准化升级
- 增加偏振滤镜、固定支架、外部供电

#### P2：六周内完成测试用例冻结
行车长稳：
- 单次 **8h**
- 累计 **>200h**
- **0–120km/h**
- **20–200Hz**
- **-10℃ / 25℃ / 45℃ / 60℃**
- HUD 导航 + 蓝牙音乐 + **1080p 录像** + 语音唤醒

泊车长稳：
- **24h 静置 + 7 天待机循环**
- 哨兵模式、充电、热浸、摘戴检测、反复开关机

#### P3：验收指标正式入库
- 黑屏 / 死机：**0 次 / 8h**
- HUD 关键信息漏显：**0 次**
- 录像中断：**0 次**
- 语音误唤醒率：**<2 次 / h**
- 系统重启：**0 次**
- 表面贴脸区温度：**<40℃**
- 待机异常唤醒：**<1 次 / 24h**
- 哨兵误触发率：**<5%**
- 低电保护触发准确率：**100%**
- 冷启动：**<5s**

#### P4：专项风险补强
- HUD 拍摄失真：增加**人眼主观评分 + 光学亮度计**
- 数据量爆炸：优先**事件触发保存**，不做全量人工回看
- 4K 连续录像：单列专项压力测试，不混入所有主链路基线

---

## 一句话决策

**V1 立即立项“拍屏 + 系统日志 + 事件触发回录”的长稳验证体系，拍屏作为必要证据层保留，但绝不允许纯拍屏方案单独承担智能骑行头盔的量产验收。**
