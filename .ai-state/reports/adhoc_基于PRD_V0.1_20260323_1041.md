# 智能骑行头盔项目 V1 PRD 综合研究报告（给歌尔）

## 一、执行摘要

基于 PRD V0.1 以及 CTO、CMO、CDO 三方分析，我的最终裁决是：**歌尔 V1 不做“炫技型智能头盔”，而做“安全可量产型智能骑行头盔”**。V1 的目标不是把所有想象中的智能功能一次性塞进头盔，而是在 **6–9 个月内完成 EVT-DVT-PVT 并具备量产条件**，建立首代产品的口碑闭环。

三方观点高度一致：V1 应聚焦 **安全可见性、稳定音频互联、基础事故救援、App 管理**。具体进入 V1 量产的功能应包括：**蓝牙 5.3 音频/通话、后置高亮 LED 尾灯、无线转向灯遥控、跌倒检测 + SOS、App 连接、USB-C 充电**；其中 **自动刹车灯、语音助手唤醒**可作为 V1 的 P1 功能。**HUD/近眼显示、摄像记录、Mesh 群组对讲、主动降噪 ANC、车机深度互联/雷达联动**应明确推迟到 V2。

关键取舍依据有六个维度：  
1. **用户价值是否高频刚需**  
2. **量产成熟度是否足够**  
3. **是否显著增加认证和结构风险**  
4. **是否拉高 BOM、功耗、重量和发热**  
5. **是否会拖累首代口碑与售后**  
6. **是否符合歌尔现阶段优势：声学、结构、电子集成、量产一致性**

竞品数据也支持这一判断。市场已验证的主流量产功能集中在 **灯光 + 蓝牙 + App + SOS**：  
- **LIVALL BH51M Neo**：蓝牙、尾灯、转向、SOS，续航约 **10h**，重量约 **480g**，价格 **￥500–900**  
- **Sena R2 EVO / M1 EVO**：蓝牙/Mesh 对讲、尾灯，续航 **12–18h**，重量约 **400–430g**，价格 **￥900–1600**  
- **Lumos Ultra / Matrix**：灯光、转向、App，续航 **4–10h**，重量约 **370–440g**，价格 **￥800–1800**  
- **Eyelights EyeRide（外挂 HUD）**：HUD、导航、通话、音乐，价格 **￥2000+（不确定）**，用户反馈集中在 **续航差、阳光下不可视、结构脆弱、安装复杂**

因此，V1 应走 **LIVALL / Lumos / Sena 的成熟路径组合**，而不是一开始走 Eyelights 式 HUD 路线。首代产品的成功标准不是“最先进”，而是“能卖、耐用、少差评、可持续迭代”。

## 二、核心分析

### 1. 产品定义与 V1 战略定位

首先纠正一个潜在偏差：CDO 分析中提到“无论面向自行车还是电助力骑行场景”，这并不构成产品定义偏离，因为仍属于“智能骑行头盔”范畴，没有改变产品品类；但在对外 PRD 中必须统一表述为 **智能骑行头盔**，避免团队内部因场景扩展而误导为其他头盔品类。

V1 的战略定位应明确为：  
**一顶每天愿意戴、具备真实安全增益、并且能稳定量产交付的智能骑行头盔。**

这意味着头盔首先是头盔，其次才是智能硬件。安全认证和佩戴体验优先级高于一切电子功能。三方都强调了这一点：  
- CDO 强调 **CPSC / EN1078** 等认证路径必须优先，电子件布局不能破坏 EPS 吸能连续性，不能形成冲击“硬点”  
- CTO 强调量产前必须完成 **CPSC / EN1078 / CE / FCC / RED** 闭环  
- CMO 强调用户购买时首先看的是 **安全/舒适 > 可靠性 > 可见性功能 > 音频互联 > 炫技显示**

因此，V1 PRD 的第一章不应写“智能功能清单”，而应写 **产品目标、适用场景、认证目标、重量目标、续航目标、可靠性目标**。

---

### 2. 哪些功能必须进 V1 量产

#### 2.1 蓝牙 5.3 音频/通话：P0，必须量产
这是三方一致支持的核心功能。原因有三：

第一，用户价值高频。导航播报、来电接听、音乐播放是骑行中最常见的连接需求。  
第二，技术成熟。CTO 给出的方案包括 **高通 QCC 系列 / Bestechnic 恒玄 / Airoha**，BOM 预估 **$8–15**，风险低，且歌尔在声学、麦克风、天线调优上有优势。  
第三，它能替代 HUD 的一部分核心价值：**“无需低头看导航”**，但实现成本和风险远低于 HUD。

建议指标保留 CTO 数据：  
- 双麦 ENC 通话降噪  
- 通话上行 SNR 目标 **>58 dB**  
- 风噪场景 **25 km/h** 下语音可懂度可接受  
- 开放式扬声器在 **30 km/h** 以上音质下降可接受，因为导航播报仍是主价值

#### 2.2 后置高亮 LED 尾灯 + 转向灯遥控：P0，必须量产
这是最直观的安全价值。CTO 建议夜间可视距离目标 **>80 m**，BOM 预估 **$3–6**。CMO 和 CDO 都认为这是最容易被用户理解和接受的功能，因为它直接提升“被看见”的概率。

建议 V1 量产定义：  
- 后置高亮 LED 尾灯  
- 常亮 / 慢闪 / 快闪  
- 左右转向灯  
- 车把遥控器控制  
- 遥控器连接稳定性目标 **>99%**  
- 防水至少 **IPX4**

#### 2.3 跌倒检测 + SOS：P0，必须量产
这是 V1 的“智能感”核心之一，也是与普通带灯头盔拉开差异的关键。CTO 建议采用 **Bosch BMI270 / ST LSM6DSOX** 级别 6 轴 IMU，BOM **$1.5–3**。  
关键不在硬件，而在算法和手机侧联动。建议保留以下指标：  
- 真跌倒召回率目标 **>90%**  
- 日常误报率目标 **<1 次/100 小时骑行**  
- 通过手机 App 发短信/推送给紧急联系人

需要在 PRD 中明确依赖条件：**手机在线、蓝牙连接、App 后台权限开启**。尤其 iOS/Android 后台策略差异大，必须在产品说明和交互设计中前置告知。

#### 2.4 App 连接 + OTA + 电量管理：P0，必须量产
App 在 V1 中不是“内容平台”，而是设备管理工具。三方都支持其作为基础能力存在。V1 App 应只承担：  
- 配对连接  
- 灯效设置  
- 紧急联系人配置  
- 电量显示  
- 固件升级 OTA

不建议 V1 强推订阅或复杂社区功能。CMO 已明确指出：首代最重要的是装机量和口碑，而不是软件变现。

#### 2.5 USB-C 充电 + 续航目标：P0，必须量产
CTO 建议电池 **1500–2000mAh**，综合续航目标 **8–12 小时**，纯灯光 **>15h**。这是合理的 V1 目标区间。若续航低于这一水平，将直接落入 Eyelights 类产品“续航差”的负面认知。

---

### 3. 哪些功能进入 V1 但降优先级

#### 3.1 自动刹车灯：P1
基于 IMU 减速度触发，用户价值明确，技术复杂度可控，但需要更多路测来降低误触发。建议作为 V1 P1：若 DVT 阶段算法稳定则上线，否则不影响主版本量产。

#### 3.2 语音助手唤醒：P1
建议仅调用手机 **Siri / Google Assistant**，不做本地 ASR。原因是风噪环境下本地语音识别稳定性差，且会增加算力、功耗和开发复杂度。

---

### 4. 哪些功能明确推迟到 V2

#### 4.1 HUD / 近眼显示：P2，推迟
这是三方最一致的结论。CTO 给出了最完整的数据支撑：  
- HUD 模组额外功耗通常 **1.5–3W**  
- 若电池仅 **1500–2000mAh**，实际续航可能掉到 **2–4 小时**  
- 阳光下可读性要求高亮微显示 + 高效率光学，成本高  
- 结构可靠性、抗震、装调一致性、视轴偏差都难控

CMO 和 CDO 也都引用了 Eyelights 的典型问题：**阳光下看不清、续航差、结构脆弱、安装复杂、像半成品**。因此 HUD 不是伪需求，而是 **不适合首代量产主项目**。

#### 4.2 摄像记录：P2，推迟
CTO 数据显示：1080p 录制 + 存储 + 散热 + 防抖，BOM 至少增加 **$15–30**，功耗增加 **1.5–2.5W**，还会引入隐私、TF 卡可靠性、发热投诉。对首代量产极不友好。

#### 4.3 Mesh 群组对讲：P2，推迟
Sena 已证明其价值，但其壁垒在协议、射频、音频并发稳定性。若做 **4 人以上稳定群组**，开发和测试复杂度显著上升。V1 先做手机通话 + 导航播报更稳。

#### 4.4 ANC、车机深度互联、雷达联动：P2，推迟
这些功能都不是 V1 的成交关键，却会显著增加系统复杂度、功耗和售后风险。

---

### 5. V1 量产约束：认证、重量、结构、可靠性

这一部分是 CDO 分析最有价值的补充，应纳入 PRD 的“非功能需求”章节。

#### 5.1 认证目标
至少按以下路径设计：  
- **CPSC / EN1078 / CE / FCC / RED**

#### 5.2 重量与重心
CTO 建议：  
- 城市款整盔目标 **<450g**  
- 若超过 **500g**，佩戴投诉概率明显上升

结合竞品：  
- LIVALL BH51M Neo：约 **480g**  
- Sena R2 EVO / M1 EVO：约 **400–430g**  
- Lumos Ultra / Matrix：约 **370–440g**

因此 V1 应把目标定在 **400–450g** 区间，且重心贴近头部中心，避免后坠和侧偏。

#### 5.3 防水与耐久
最低目标 **IPX4**。量产前必须完成：  
- 风噪测试  
- 跌落测试  
- 振动测试  
- 汗液腐蚀测试  
- 充电口寿命测试

## 三、冲突与裁决

### 冲突点 1：V1 是否要做更强“智能感”
- **CMO 观点**：担心 V1 过于基础，智能感不足，溢价天花板有限  
- **CTO / CDO 观点**：首代必须克制，先做安全可量产平台

**裁决**：以 CTO / CDO 为主，但吸收 CMO 的担忧。  
**理由**：V1 的智能感不靠 HUD 来建立，而靠 **跌倒 SOS + 蓝牙导航播报 + App + OTA** 来建立。这样既能形成“不是普通带灯头盔”的认知，又不引入过高量产风险。

### 冲突点 2：V1 是否做双版本定价
- **CMO 观点**：建议双版本  
  - 标准版 **699–899 元**：尾灯/转向/SOS/App  
  - Pro 版 **999–1299 元**：增加开放式音频、通话、导航播报
- **CTO / CDO 观点**：未明确双版本，但强调平台化和模块化

**裁决**：支持双版本策略，但前提是 **结构平台共用率高**。  
**理由**：双版本有助于控制首发价格门槛，同时验证音频功能的真实付费意愿；但不能做成两套完全不同结构，否则会拖慢量产。建议共用壳体、电池仓、灯组、主板平台，通过音频模组和软件能力区分标准版/Pro 版。

### 冲突点 3：自动刹车灯是否必须首发
- **CTO**：建议进入 V1 但降优先级  
- **CMO / CDO**：未强推

**裁决**：定义为 **V1 P1，可灰度上线**。  
**理由**：用户价值明确，但误触发会伤害体验。应以路测数据为准，不作为量产上市的阻塞项。

## 四、明确结论与行动建议

### 1. 给歌尔的 V1 PRD 大纲建议

1）**产品目标与定位**  
- 智能骑行头盔  
- 核心价值：提升被看见、减少低头操作、提供基础通讯与事故告警  
- 项目目标：**6–9 个月完成 EVT-DVT-PVT 并具备量产条件**

2）**目标用户与场景**  
- 城市通勤  
- 夜骑  
- 电助力骑行  
- 中短途休闲骑行

3）**P0 功能定义**  
- 蓝牙 5.3 音频/通话  
- 双麦 ENC  
- 后置高亮 LED 尾灯  
- 左右转向灯 + 车把遥控器  
- 跌倒检测 + SOS  
- App 管理  
- OTA  
- USB-C 充电

4）**P1 功能定义**  
- 自动刹车灯  
- 语音助手唤醒（Siri / Google Assistant）

5）**P2 / V2 预研功能**  
- HUD / 近眼显示  
- 摄像记录  
- Mesh 群组对讲  
- ANC  
- 车机深度互联 / 雷达联动

6）**硬件配置建议**  
- SoC：QCC / 恒玄 / Airoha  
- 传感器：6 轴 IMU  
- 音频：2 扬声器 + 2 麦  
- 电池：**1500–2000mAh**

7）**关键指标**  
- 综合续航 **8–12h**  
- 纯灯光 **>15h**  
- 夜间可视距离 **>80m**  
- 遥控器连接稳定性 **>99%**  
- 跌倒召回率 **>90%**  
- 误报率 **<1 次/100 小时骑行**  
- 重量目标 **<450g**，上限不超过 **500g**  
- 防水 **IPX4**

8）**认证与测试**  
- CPSC / EN1078 / CE / FCC / RED  
- 风噪、跌落、振动、汗液腐蚀、充电口寿命测试

---

### 2. 近期行动建议

**第一阶段：2–4 周内完成 PRD V1.0 冻结**  
明确 P0/P1/P2 边界，禁止 HUD、摄像、Mesh 进入 V1 主项目。

**第二阶段：并行启动 ID/结构/电子预研**  
重点验证：  
- EPS 吸能连续性  
- 电池仓与灯组布局  
- 重量与重心  
- 开放式音频风噪表现

**第三阶段：200–500 台工程样机内测**  
重点看：  
- 30 天留存使用率  
- 退货原因  
- NPS  
- 跌倒误报率  
- 蓝牙断连率  
- 遥控器稳定性

**第四阶段：HUD 单独立项做 V2 技术预研**  
只验证三件事：  
- 强光可视性  
- 续航  
- 抗振与装调一致性  
不与 V1 量产节奏绑定。

---

## 一句话决策

**V1 量产只做“蓝牙音频/通话 + 尾灯/转向 + 跌倒 SOS + App/OTA”的安全可量产智能骑行头盔，HUD、摄像、Mesh 全部推迟到 V2。**
