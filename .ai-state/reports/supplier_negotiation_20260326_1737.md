# 智能骑行头盔项目：歌尔 JDM 合作模式与商务条款拆解研究报告

## 一、执行摘要

本次研究的核心问题不是“歌尔能不能做”，而是**我们应当以什么条款结构与歌尔谈，才能在控制前期现金流的同时，避免后期被供应链锁死**。从第一性原理看，这个项目并非普通消费电子 ODM，而是一个同时受**头盔本体认证、HUD 光学、影像、语音声学、安全算法**五大系统耦合约束的复杂 JDM 项目。若沿用耳机、手表类项目的惯常谈法——例如总 NRE 一口价、IP 模糊归属、模具默认供应商持有、MOQ 整机一刀切——最终大概率会出现三个问题：前期投入失控、量产议价权丧失、二供切换几乎不可能。

基于 CTO 视角的分析，我的裁决是：**与歌尔应按 JDM 模式推进，但必须把 NRE、IP、模具、MOQ 四件事拆开谈、分层谈、写死谈。**  
其中最关键的不是价格，而是控制权边界。

围绕用户已定义的产品方向——智能骑行头盔——团队分析没有偏离产品品类，方向上是正确的。但 CTO 分析中夹带了较多 V1 功能优先级建议，例如 HUD 先做基础导航、4K 录像先降到 1080P/2K 稳定量产、ADAS 放到 V2。这些建议本身合理，但要警惕它们是在回答“怎么降低技术风险”，而不是直接回答“歌尔合作模式与商务条款怎么谈”。因此本报告会保留这些技术判断作为商务谈判的前提条件，而不会让其喧宾夺主。

综合结论如下：

1. **NRE 付款方式**：不接受 30/40/30 或 40/30/30 的传统强供应商结构，优先谈 **20/30/30/20**，备选 **15/30/30/25**。核心逻辑是把第一笔钱买成“技术真相”，把后续付款绑定 EVT、DVT、PVT 的明确交付物。
2. **IP 归属**：必须拆成**背景 IP、前景 IP、客户专属定制 IP**。歌尔既有平台、算法库、协议栈、工艺方法归歌尔；头盔外观 ID、UI/HUD 信息架构、事故判定规则、车队交互逻辑、App 数据结构与用户资产归我方；项目中形成的客户专属成果应归我方所有，歌尔不得向第三方复用。
3. **模具归属**：原则上**谁出钱谁拥有，歌尔保管**。外观模具、专属结构模具、EPS/EPP 缓冲层模具、面罩/导光件/支架模具等必须归我方；通用工装和底层测试夹具可归歌尔，但我方需保留项目使用权。
4. **MOQ 判断**：这不是普通蓝牙设备，不能幻想 1k–3k 台就拿到好条件。合理预期是：EVT/DVT 10–50 台，PVT 200–500 台，量产首单 **5k–10k 台**，更优是签年度框架 **10k–20k 台** 分批拉货。低于 **3k 台**，歌尔大概率会通过单价、NRE 或资源优先级把成本转回来。
5. **行业惯例**：行业里确实常见供应商保留背景 IP、客户支付 NRE、模具代管、MOQ 与价格联动，但对我们这种“头盔本体 + 消费电子”复合项目，**不能照搬普通 ODM 惯例**，必须在合同中增加退出机制、二供切换权、KPI 未达成时的资料与模具转移权。

一句话说，**这次谈判的本质不是压价，而是用条款换控制权，用分阶段付款换决策权，用 IP 和模具归属换未来议价权。**

## 二、核心分析

### 1. 为什么这个项目不能按普通 ODM 条款谈

CTO 的判断是对的：该项目不是标准蓝牙耳机项目，而是一个高耦合系统工程。V1 定义中已包含以下关键硬件与指标：

- **HUD**：入眼亮度 **>3000 nits**，参考 EyeRide 公开参数 **20° FOV / 720p / 3000 nits / $500–700 零售价**
- **影像**：4K 摄像，传感器目标 **≥1/1.56" IMX 大底**
- **音频**：**40–53mm** 动圈，**3–5 麦阵列**，ANC/ENC
- **连接**：**Wi‑Fi 6 + BT 5.2/5.3**
- **续航**：**≥3500mAh**，目标 **8h（HUD 导航 + 蓝牙音乐 + 1080P 录像）**
- **整机重量**：**≤1.65kg**
- **认证**：**ECE 22.06 / DOT / 3C + FCC / CE-RED / SRRC + UN38.3**

这些参数意味着，歌尔面对的不是单一电子模组开发，而是“受法规约束的防护件 + 高集成消费电子”的联合开发。其额外复杂度来自：

1. 头盔壳体、缓冲层、结构件开发；
2. HUD 光机与视轴校准；
3. 高速风噪语音与麦克风阵列调优；
4. 防水、防震、热管理；
5. 整机认证联动。

因此，若按普通 ODM 方式谈“一口价总 NRE + 量产再说”，本质上是在为供应商转嫁不确定性买单。我们真正要解决的问题不是“拿到一个报价”，而是**把不确定性拆成可管理的里程碑**。

---

### 2. NRE 怎么分阶段付：必须买“技术真相”，不是先付大额学费

CTO 建议采用四阶段付款，而不是两阶段或简单三段式。我支持这个判断，并将其作为正式裁决。

#### 推荐结构：20/30/30/20  
备选：15/30/30/25

#### 阶段 A：可行性 / NPI 立项（15%–20%）
交付物包括：
- 系统架构；
- HUD / 摄像 / 声学 / 电池热设计可行性报告；
- 重量预算、功耗预算、认证风险清单；
- EVT 样机 BOM 上限。

这一阶段必须单列，因为我们最大的风险不是“能不能做出样机”，而是**能不能在 ≤1.65kg、8h、ECE 22.06 约束下同时成立**。这笔钱买的是技术真相，而不是 PPT。

#### 阶段 B：EVT 工程样（25%–30%）
交付物包括：
- 功能样机；
- HUD 可视性验证；
- 语音识别风噪测试；
- 影像 EIS/WDR 初版；
- 热仿真与初测。

这里的生死线很明确：**100km/h 风噪下语音识别可用、HUD 在强日照下可读、摄像头在震动场景可用**。若这三项不过，继续投入就是典型 XY 问题——在错误方向上优化细节。

#### 阶段 C：DVT 设计验证（25%–30%）
交付物包括：
- DVT 样机；
- 可靠性测试；
- 防水、跌落、振动、温升；
- 认证预测试；
- 试产工装方案。

#### 阶段 D：PVT / 量产导入（20%–30%）
交付物包括：
- 小批量试产；
- 良率报告；
- 测试治具；
- 量产 SOP；
- 认证送检版本冻结。

#### 行业惯例与我们的裁决
行业常见是 **30/40/30** 或 **40/30/30**。但这类结构对强势供应商有利，对新品牌极不友好，因为前期付款过重，而关键风险尚未被验证。  
**裁决：坚持 20/30/30/20，最多退让到 15/30/30/25，不接受前两阶段合计超过 50%。**

---

### 3. IP 归属怎么约定：必须分层，不要幻想“全部归我”

IP 是本项目最容易被谈糊、也最容易埋雷的部分。CTO 的分层方法正确，我进一步明确如下。

#### 3.1 背景 IP（归歌尔）
包括：
- 既有声学算法库；
- HUD 通用光机平台；
- 通用 BT / Wi‑Fi 协议栈；
- 通用生产工艺、测试治具底层方法。

这是行业惯例，归供应商所有，客户获得项目内使用许可。  
如果我们试图把这些全部拿走，只会导致 NRE 暴涨，且歌尔会在合作意愿上明显下降。

#### 3.2 前景 IP（按层拆）
**必须归我方拥有或独占的内容：**
- 头盔外观 ID；
- UI / HUD 信息架构；
- 场景联动逻辑；
- 事故判定规则；
- 车队交互逻辑；
- App 数据结构与用户资产。

**可共享或许可的内容：**
- 基于歌尔平台做的硬件适配；
- 通用驱动；
- 通用声学调参框架；
- 通用 EIS pipeline。

#### 3.3 客户专属定制 IP
建议合同写法采用 CTO 提出的思路：

> 客户定制成果（Customer-specific Foreground IP）归客户所有；供应商保留其背景 IP 及通用能力，但不得向第三方复用客户专属外观、结构布局、UI、交互逻辑及专属算法参数。

这是最实用的写法。因为真正的风险不是歌尔保留底层平台，而是它未来拿我们的项目成果做“换壳版”卖给别人。

---

### 4. 模具归谁：必须写死，不然二供切换是空话

模具问题不能模糊处理。  
本项目中的“模具”不只是注塑件，还可能包括：

- 外壳复材工装；
- EPS / EPP 缓冲层模具；
- 面罩 / 导光件 / 支架模具；
- 防水胶路治具；
- 光学校准夹具。

#### 裁决原则
1. **外观模具 / 专属结构模具：我方出资，我方所有；**
2. **通用工装 / 测试夹具：歌尔所有，但我方享项目使用权；**
3. **若歌尔垫资模具：达到累计采购量后自动转移所有权。**

这里必须增加一条：若歌尔连续未达交付 KPI、质量 KPI 或认证节点 KPI，我方有权带走专属模具和专属设计资料，切换第二供应商。否则“模具归我方”只是纸面权利，实际仍被代管关系锁死。

---

### 5. MOQ 多少合理：按阶段拆，不要整机一刀切

CTO 给出的判断是：

- **EVT / DVT 样机：10–50 台**
- **PVT 试产：200–500 台**
- **量产首单：5k 台起谈**
- **更优目标：年度框架 10k–20k 台，分批拉货**

我认可这个区间。  
原因很简单：项目包含头盔壳体、复材工艺、HUD 模组、多认证、复杂装配，经济批量天然高于普通消费电子。若我们执着于极低 MOQ，本质上是在解决错误问题——表面上是“降低首单压力”，实际上会换来更高单价、更差资源优先级、更弱交付承诺。

#### 行业惯例判断
普通消费电子 ODM 的 MOQ 常见可做到 **3k–5k 台**；  
但对本项目，**V1 合理 MOQ 是首单 5k–10k 台**。  
低于 **3k 台**，歌尔大概率会通过报价结构把损失补回来。

---

### 6. 技术边界对商务条款的影响

虽然本次任务重点是商务条款，但技术边界直接决定谈判筹码。CTO 的 V1/V2 建议应作为商务前提保留：

#### V1 必须锁定
1. **HUD 基础导航 + 安全提示**  
参考 EyeRide：**720p / 3000 nits / 20° FOV**。  
V1 不追求复杂 AR SLAM，先做固定视场导航 + 告警光效。

2. **语音交互**  
**3–5 麦阵列 + 本地唤醒**。  
100km/h 可用依赖麦位设计 + 风噪模型，不是只靠算法。

3. **1080P / 2K 稳定录像先量产**  
4K 可保留硬件能力，但 V1 先保证热和续航。

4. **SOS 事故判定**  
先用 IMU + GPS + 佩戴检测做规则引擎，V2 再加模型优化。

#### V2 建议增强
1. **ADAS 前向 / 碰撞预警**
2. **车队 Mesh**
3. **全息信息流 / 社交成长**

这些建议的商务意义在于：**合同中必须允许 EVT 后缩减 V1 范围并二次报价**。否则歌尔会按全量复杂度收费，而我们失去产品收敛权。

## 三、冲突与裁决

本轮只有 CTO 一个角色，没有显性跨角色冲突，但存在两个潜在偏移点，我在此明确裁决。

### 冲突点 1：技术方案建议是否偏离本次商务研究任务
- **表现**：CTO 花了较多篇幅讨论 V1/V2 功能取舍，如 4K 降到 1080P/2K、ADAS 放 V2。
- **裁决**：这些内容不构成偏离产品定义，但属于“支撑商务判断的前提”，不能替代商务条款本身。保留其结论，用于定义里程碑验收标准和退出机制。

### 冲突点 2：是否应争取全部 IP
- **表现**：团队容易天然倾向“既然付了 NRE，所有 IP 都该归我”。
- **裁决**：这是典型非第一性原理思维。真正要争的不是全部 IP，而是**客户专属差异化资产和二供切换能力**。背景 IP 全拿既不现实，也不经济。应采用“背景 IP 归歌尔、客户专属前景 IP 归我方、通用改进项许可共享”的结构。

## 四、明确结论与行动建议

### 明确结论
1. **合作模式**：与歌尔谈 **JDM**，不是纯 ODM。
2. **NRE 付款**：主张 **20/30/30/20**，备选 **15/30/30/25**。
3. **IP 归属**：  
   - 背景 IP 归歌尔；  
   - 头盔外观 ID、UI/HUD 信息架构、事故判定规则、车队交互逻辑、App 数据结构与用户资产归我方；  
   - 客户专属定制成果归我方，歌尔不得向第三方复用。
4. **模具归属**：专属模具我方所有、歌尔代管；通用工装归歌尔但我方有项目使用权；垫资模具达到采购量后自动转移。
5. **MOQ**：  
   - EVT/DVT：**10–50 台**  
   - PVT：**200–500 台**  
   - 首单量产：**5k–10k 台**  
   - 年度框架：**10k–20k 台**
6. **行业惯例**：可参考，但不能照搬普通消费电子 ODM 惯例；必须增加退出机制、KPI 绑定、二供切换权。
7. **仍需正式核实的信息**：  
   - 歌尔对该类“头盔 + HUD + 影像”项目的真实 NRE 报价区间；  
   - 是否接受客户拥有专属前景 IP；  
   - 最低经济 MOQ；  
   - 是否愿意承接 **ECE 22.06** 约束下的整盔 JDM 责任边界。

### 行动建议
1. 立即输出一版对歌尔的 **RFI/RFQ 清单**，要求其按四阶段 NRE 分项报价，而非总包价。
2. 法务与采购联合起草 **IP 分层条款模板** 与 **模具归属条款模板**。
3. 在商务谈判前，内部先冻结 V1 验收指标：  
   - **>3000 nits HUD 可读性**  
   - **100km/h 风噪语音可用**  
   - **1080P/2K 稳定录像**  
   - **≤1.65kg**  
   - **≥3500mAh、8h 目标**
4. 合同中加入 **EVT 后缩范围权**、**KPI 未达成退出权**、**专属资料与模具转移权**。
5. 以 **5k 台首单、10k–20k 年框** 为谈判基线，换取更优单价与资源优先级。

## 一句话决策

**与歌尔按 JDM 推进，但必须坚持“20/30/30/20 分阶段 NRE + 背景/前景/专属 IP 分层归属 + 专属模具我方所有 + 首单 5k 起”的条款框架，否则宁可不签。**
