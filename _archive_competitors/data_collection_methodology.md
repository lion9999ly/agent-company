## 二、反爬应对策略分级

### Level 1: 无反爬/弱反爬

**特征**: 直接访问返回完整内容
**方案**: `requests + BeautifulSoup`

```python
import requests
from bs4 import BeautifulSoup

headers = {'User-Agent': 'Mozilla/5.0...'}
r = requests.get(url, headers=headers)
soup = BeautifulSoup(r.text, 'html.parser')
```

**适用**: 企业官网、政府网站、部分媒体

### Level 2: 动态渲染

**特征**: HTML中无内容，需JS执行
**方案**: `DrissionPage / Selenium`

```python
from DrissionPage import ChromiumPage
page = ChromiumPage()
page.get(url)
time.sleep(3)  # 等待渲染
content = page.ele('css:.content').text
```

**适用**: React/Vue单页应用、小红书、部分媒体

### Level 3: 登录墙

**特征**: 未登录跳转到登录页
**方案**: 复用用户浏览器

```bash
# 启动Chrome远程调试
chrome.exe --remote-debugging-port=9222 --user-data-dir="..."
```

**适用**: 京东、天猫、淘宝

### Level 4: 强反爬

**特征**: 多重验证（验证码/滑块/行为检测）
**方案**: 手动采集

**适用**: 淘宝详情页、京东登录后评价

| 等级 | 来源类型 | 示例 | 可信度 |
|------|----------|------|--------|
| **high** | 官方数据 | 官网规格、产品包装 | 最高 |

## 四、采集流程标准化

### 4.1 前置检查

```
1. 检查目标网站是否需要登录
2. 检查是否有公开API
3. 检查robots.txt
4. 评估法律合规性
```

### 4.2 工具选择决策树

```
开始
 ↓
是否需要登录？ ─Yes→ 用户浏览器复用 / 手动采集
 ↓ No
是否动态渲染？ ─Yes→ DrissionPage / Selenium
 ↓ No
requests + BeautifulSoup
```

### 4.3 数据验证

```
1. 检查数据完整性
2. 与官方数据交叉验证
3. 记录来源URL和采集时间
4. 标注置信度等级
```

## 五、常见问题与解决方案

### Q1: requests返回空内容

**原因**: 动态渲染
**解决**: 使用DrissionPage

### Q2: 412/403错误

**原因**: 被识别为爬虫
**解决**:
1. 添加随机User-Agent
2. 添加随机延迟
3. 使用代理IP
4. 复用用户浏览器

### Q3: 需要登录才能访问

**原因**: 登录墙
**解决**:
1. 检查是否有公开API
2. 复用已登录浏览器
3. 手动采集

### Q4: 数据提取不准确

**原因**: 选择器不匹配
**解决**:
1. 检查HTML结构
2. 使用多个备选选择器
3. 关键词匹配作为兜底

## 八、本次采集实战记录 (2026-03-16)

### 成功数据源

| 数据源 | 成功率 | 获取数据 | 关键技术 |
|--------|--------|----------|----------|
| INMO官网 (inmoxr.com) | ✅ 100% | 15项完整规格 | requests + BeautifulSoup |
| Xreal官网 (xreal.com) | ✅ 80% | HTML (JS渲染难提取) | requests |
| TCL美国站 | ✅ 60% | 产品页面 | requests |
| B站视频页 | ✅ 90% | 播放量、点赞、标题 | requests + 正则 |
| 微博搜索页 | ✅ 70% | 页面HTML | requests |
| 抖音搜索页 | ✅ 70% | 页面HTML | requests |

### 失败数据源

| 数据源 | 失败原因 | 错误码 |
|--------|----------|--------|
| 京东商品页 | 登录墙+强反爬 | 200→登录页 |
| 天猫店铺 | 登录墙 | 200→登录页 |
| 什么值得买 | 搜索需登录 | 202/404 |
| 慢慢买比价 | 404错误 | 404 |
| B站搜索API | 412反爬 | 412 |
| 知乎搜索 | 403禁止 | 403 |
| Reddit | 403禁止 | 403 |
| Amazon | 503服务不可用 | 503 |

### 关键发现

1. **官网最可靠** - 企业官网反爬最弱，数据最权威
2. **海外站更开放** - xreal.com、global.rokid.com比国内站友好
3. **B站视频页可直接访问** - 不需要API，直接解析HTML
4. **电商平台最难攻克** - 京东/天猫的登录墙几乎无法绕过

### 工具性能对比

| 工具 | 成功场景 | 失败场景 |
|------|----------|----------|
| requests | 静态官网、视频页 | 电商、需登录站点 |
| DrissionPage | 动态渲染页面 | 电商登录墙 |
| browser-use | - | Windows兼容问题 |

### 最佳实践更新

1. **首选官网** → requests直接获取
2. **次选海外站** → 全球化版本通常更开放
3. **视频平台** → 直接访问视频页面解析
4. **电商数据** → 需要用户配合或第三方API
