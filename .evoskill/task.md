# Task: HUD Demo Validation

验证生成的 HUD Demo HTML 文件是否符合 skills/hud-demo/SKILL.md v2.0 的 14 条验收标准。

## 输入
- demo_outputs/hud_demo_v2.html 文件路径

## 输出
- 每条验收标准的通过/失败判定
- 总体通过率

---

# 14 条验收标准

## AC1: 光学模式编码差异
三种光学模式切换后，同一个 ADAS 预警的编码方式必须可见差异。
- 全彩模式（FreeForm/Waveguide）：使用颜色编码（红/橙/黄）
- 单绿模式：使用亮度编码（100%/80%/50%/20%）
- 检验方法：触发 FCW 预警，切换 F1/F2/F3，观察编码变化

## AC2: 单绿模式色相唯一
单绿模式下不得出现 #00FF00 以外的任何色相。
- 所有图标、文字、边框都必须是 #00FF00
- 包括 ⚠️ 图标在内的所有元素
- 检验方法：切换到单绿模式，遍历所有 UI 元素，检查颜色值

## AC3: 中央透明区域
中央区域必须完全透明，可见路面背景。
- 中央不得放置任何 HUD 信息元素
- 检验方法：检查中央区域是否有 DOM 元素

## AC4: ADAS 信息完整性
ADAS 预警必须同时显示：
- 距离（米）
- 关闭速度（km/h）
- TTC（秒，倒计时）
- 检验方法：触发预警，检查显示内容

## AC5: 多重预警优先级
多重预警并发时：
- TTC 最短的威胁最大化显示
- 其他威胁弱化但可见（缩小、降低亮度）
- 检验方法：同时触发 FCW + BSD，观察显示优先级

## AC6: BSD 外边缘闪烁
BSD 左后方来车时：
- 闪烁出现在左侧竖条的**外边缘**（左侧条的左边）
- 不是内边缘
- 检验方法：触发 BSD 左后方预警，检查闪烁位置

## AC7: LDW 内边缘闪烁
LDW 车道偏离时：
- 左侧压线 → 左侧竖条**内边缘**虚线闪烁
- 右侧压线 → 右侧竖条**内边缘**虚线闪烁
- 检验方法：触发 LDW 左/右，检查闪烁位置

## AC8: 优先级抢占保留
高优先级触发时：
- 低优先级信息缩小到角落
- 不消失，仍可见
- 检验方法：导航模式下触发来电，检查导航是否缩小但可见

## AC9: 预警后自动恢复
预警结束后：
- 自动恢复之前的状态
- 无需用户操作
- 检验方法：导航 → 预警 → 预警结束，检查是否恢复导航

## AC10: touring 语音标记
touring 剧本（60s）必须包含：
- 语音标记精彩瞬间
- 显示 "Moment saved!" 或类似字幕
- 检验方法：播放 touring 剧本，检查语音标记事件

## AC11: 语音字幕位置
语音交互字幕必须：
- 出现在非可视区域（画布底部或顶部）
- 不是 HUD 显示区域
- 检验方法：触发语音交互，检查字幕位置

## AC12: 背景透明度可调
路面背景透明度必须：
- 提供 sandbox 控件调节
- 实时生效
- 检验方法：调节透明度滑块，观察背景变化

## AC13: 交通标志持续显示
交通标志识别必须：
- 持续显示在 RT zone
- 直到下一个标志才替换
- 检验方法：触发标志 A → 标志 B，检查替换行为

## AC14: 显示区域可调
显示区域大小和位置必须：
- 提供 sandbox 控件调节
- 包括宽度、高度、位置偏移
- 检验方法：调节参数，观察布局变化

# 验证约束

- 每条标准必须独立检验
- 通过标准：完全符合描述
- 失败标准：部分或全部不符合
- 总体通过率 = 通过数 / 14

# Constraints

- 不得简化或省略任何验收标准
- 必须逐条检验，生成详细报告
- 检验结果必须可追溯（记录具体观察值）
