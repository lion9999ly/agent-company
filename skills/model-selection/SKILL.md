# Model Selection Skill

## 核心原则

> **多模型是补丁，不是默认。**

默认使用最强单模型（当前：`o3-deep-research` 或 `gpt-5.4`）。只有在特定条件下才启用多模型阵列。

---

## 五种启用多模型的条件

| 条件 | 检测方法 | 启用策略 |
|------|----------|----------|
| **上下文 >60% 导致降智** | Token 计数 > 最大上下文 60% | 切换长上下文模型或拆分任务 |
| **输出 >4000 token 导致后半段降智** | 生成内容超过 4000 token | 分段生成，每段独立评审 |
| **需要不同工具能力** | 任务需要特定工具（如搜索、视觉） | 选择具备该工具能力的模型 |
| **单模型连续两次同类错误** | 相同错误重复出现 2 次 | 切换不同架构模型（如 GPT→Gemini） |
| **需要对抗偏见** | 评审/决策类任务 | 异构模型交叉验证（≥2 模型） |

## 决策流程图

```

    │
    ▼

    │
    ├─ ≤60% ──────────────────────┐
    │                              │
    ▼                              ▼
估算输出长度                      使用最强单模型
    │                              │
    ├─ ≤4000 token ────────────────┤
    │                              │
    ▼                              │
检查是否需要特殊工具               │
    │                              │
    ├─ 不需要 ─────────────────────┤
    │                              │

    │                              │
    ├─ 不是 ──────────────────────┘
    │
    ▼

```

## 强制声明规则

每次启用多模型时，必须在代码/日志中声明：

```python
# 示例
result = call_multi_model(
    models=["gpt_5_4", "gemini_2_5_flash"],
    reason="评审任务需要异构模型交叉验证（对抗偏见）",
    task_type="review"
)
```

**声明格式**：
- `reason`: 为什么启用多模型（必须来自五种条件之一）
- `task_type`: 任务类型
- `expected_benefit`: 预期收益

---

## 不需要多模型的场景

| 场景 | 推荐 |
|------|------|
| 简单问答 | 单模型（gemini_2_5_flash） |
| 代码生成 | 单模型（gpt_5_4） |
| 文档整理 | 单模型（doubao_seed_pro） |
| 数据提取 | 单模型（任意） |
| 格式转换 | 单模型（任意） |

---

## 模型能力矩阵

| 模型 | 上下文 | 工具能力 | 适用场景 |
|------|--------|----------|----------|
| o3-deep-research | 16000 | 深度搜索 | 研究任务（默认最强） |
| gpt-5.4 | 无限 | 代码/推理 | 复杂推理、架构设计 |
| doubao_seed_pro | 4096 | 中文优化 | 中文任务、评审 |
| gemini_2.5_flash | 8192 | 快速 | 快速响应、轻量任务 |

## 示例：正确使用

```python
# ✅ 正确：评审任务，启用多模型对抗偏见
def review_proposal(proposal):
    return call_multi_model(
        models=["gpt_5_4", "doubao_seed_pro"],
        reason="评审任务需要异构模型交叉验证（对抗偏见）",
        task_type="review"
    )

# ❌ 错误：简单问答启用多模型
def quick_qa(question):
    return call_multi_model(models=["gpt_5_4", "gemini_2.5_flash"])  # 不声明 reason
```

---

## 检查清单

每次调用前检查：
1. 上下文是否 >60%？
2. 输出是否预计 >4000 token？
3. 是否需要特殊工具？
4. 是否已连续出错 2 次？
5. 是否评审/决策任务？

**如果全部为"否"，使用单模型。**
