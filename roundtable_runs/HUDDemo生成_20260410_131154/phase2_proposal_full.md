# HUD Demo 视觉与交互设计方案

## 方案描述

本方案旨在创建一个零依赖、单HTML文件的HUD演示系统，精准传达“被保护着”的核心产品价值。方案严格遵循“四角布局、中央留空”的安全铁律，并构建了一套完整的视觉语义与状态优先级系统。

**1. 视觉骨架与信息架构**
- [偏好·--] 采用 **“四角锚定 + 底部主信息区”** 的混合骨架。四角（LT/RT/LB/RB）专用于ADAS预警方位指示和极简状态（如连接、电量）。底部中央区域（下缘）作为**主信息区**，承载导航、速度、音乐等核心信息流。中央视野绝对留空。
- 此设计回应了约束 **[CDO-1]**（四角布局中央留空）与 **[CTO-7]**（右下/下缘主信息区），同时融合了 **[CMO-3]** 的极简安全导向。底部主信息区借鉴了 **[CDO-8]** 中“2-3米外7英寸屏幕”的视觉参考，确保一瞥可读。

**2. 状态系统与优先级模型**
- [判断·高] 定义 **7个核心页面态**，并非全屏页面，而是基于一个持续渲染的“骑行主态”进行叠加的图层。状态按优先级排序为：`紧急预警 (S-EMERGENCY)` > `来电/通信 (S-CALL)` > `导航引导 (S-NAV)` > `主动语音交互 (S-VOICE)` > `音乐信息 (S-MUSIC)` > `系统通知 (S-NOTICE)` > `骑行主态 (S-RIDING)`。
- 高优先级状态会以半透明蒙层方式部分覆盖低优先级信息的显示区域，但**绝不侵入中央视野**。此模型回应了 **[CTO-8]** 和 **[CTO-11]**，确保并发事件处理符合真实HUD交互逻辑。

**3. 全彩功能色语义系统**
- [判断·中] 建立严格的颜色编码体系，作为信息识别的第一维度：
    - **速度/基础信息：白色 (#FFFFFF)** - 中性，高可读性。
    - **导航：蓝色 (#4A90E2)** - 指引、信任感。
    - **ADAS预警：红色 (#FF3B30)** - 危险、最高优先级，配合**对应方位四角的脉冲式高亮闪烁**。
    - **Mesh组队：青色 (#5AC8FA)** - 连接、队友。
    - **音乐/媒体：紫色 (#AF52DE)** - 娱乐、放松。
- 此系统直接回应 **[CDO-6]**，并强化了 **[CDO-7]**（冲击力仅用于预警）。预警时，对应方位角（如左后盲区预警闪烁LB区域）的红色脉冲是 **[CDO-2]** 所要求的空间化表达的核心。

**4. 演示叙事与自动剧本**
- [判断·中] 设计一个约 **90秒的自动演示剧本**，节奏为：开机自检 → 日常骑行（速度、音乐） → 触发导航（展示逐向引导） → 模拟Mesh组队通话切入 → 连续触发三个ADAS预警场景（右后盲区、左侧开门杀、前车碰撞）作为高潮 → 恢复平静，展示语音设置目的地 → 结束。
- 剧本通过底部时间轴控制，并可与键盘操作无缝切换。这满足了 **[CMO-4]**（3分钟内）和 **[CMO-5]**（展示核心ADAS场景）的要求，并通过 **[CTO-9]** 确保状态机一致。

**5. 技术实现与视觉降噪**
- [判断·高] 使用纯原生HTML/CSS/JS实现，所有代码、样式、资源（Base64图标/字体）内联于单个HTML文件，满足 **[CTO-1]** 和 **[CTO-10]**。
- 实现 **S0-S3速度分级信息密度控制**（S0静止显示全部，S3高速仅保留速度与预警），以及 **全局“全彩/单绿”视觉方案切换**，以模拟OLED与光波导两种光学方案的显示差异。这直接回应 **[CDO-4]**。
- 所有交互（剧本控制、状态切换、方案切换）通过键盘事件全覆盖，满足 **[CTO-2]** 和 **[CTO-13]** 的演示可控性要求。

## 决策点回应
- **决策1：采用“四角锚定+底部主信息区”混合骨架** → 回应约束：[CDO-1], [CTO-7], [CMO-3]（置信度：[判断·中] - 基于类似HUD设计范式与安全原则的平衡）
- **决策2：定义基于优先级的7层叠加状态模型** → 回应约束：[CTO-8], [CTO-11], [CDO-5]（置信度：[判断·高] - 基于HUD/车机HMI状态建模的直接经验）
- **决策3：建立全彩功能色语义与ADAS空间化预警** → 回应约束：[CDO-2], [CDO-6], [CDO-7], [CMO-1]（置信度：[判断·中] - 基于航空/汽车仪表设计规范类推）
- **决策4：设计以ADAS预警为高潮的90秒自动剧本** → 回应约束：[CMO-4], [CMO-5], [CTO-9], [CTO-5]（置信度：[判断·中] - 基于演示叙事节奏与产品核心价值的结合）
- **决策5：使用纯原生技术栈实现单文件，包含视觉降噪与方案切换** → 回应约束：[CTO-1], [CTO-10], [CDO-4]（置信度：[判断·高] - 基于零依赖演示系统的直接工程实践）

## 自评：最大风险
**最大风险在于“四角布局”与“信息一瞥可读”在高速场景下的平衡。** 将主信息区置于底部下缘，虽符合研究建议，但在高速骑行（S3状态）时，骑手视线下移的幅度和时长仍需真实环境验证。若光学方案的虚像距离（VD）不够远，或眼盒（Eye Box）不稳定，可能导致底部信息阅读困难，反而违背了“余光扫视”的原则 **[CDO-5]**。这是一个必须通过光学原型进行用户体验测试的关键点，而Demo仅能进行理想化表达。
