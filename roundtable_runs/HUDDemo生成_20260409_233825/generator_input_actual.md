# Generator Input

## Final Proposal
## 方案描述

本方案旨在交付一个符合所有技术、体验与演示约束的智能骑行头盔 HUD 交互演示原型。它是一个单一、离线、可双击运行的 HTML 文件，通过视觉、交互与叙事，向供应商与投资人传达“消灭停车掏手机”与“被保护着”的核心产品价值。

**1. 核心架构与状态管理**
- [偏好·--] 采用“三层状态机”模型，而非单一优先级列表，以清晰管理状态冲突与叠加。该模型包括：
    1.  **基础态**：常驻信息层，即“主界面”，包含速度、时间、连接状态等。
    2.  **覆盖态**：任务信息层，如“导航”、“组队”、“音乐”页面。它们覆盖在基础态之上，但彼此互斥。
    3.  **瞬时态**：最高优先级层，如“ADAS预警”、“来电”。它们强制覆盖所有其他状态，具有明确的进入和退出动画，确保即时吸引注意力。
- 此设计直接回应了状态机需清晰、可管理且能体现严格优先级（预警最高）的要求。所有页面态（7个）均通过此三层模型进行渲染，键盘与自动剧本仅需派发状态事件，由统一的状态机逻辑处理渲染。

**2. 视觉布局与信息架构**
- [偏好·--] 严格遵守 **LT/RT/LB/RB 四角布局规范**，中央视野绝对留空，确保骑行安全。
    - **LB（左下）**：主信息区。常驻显示**速度**（大字体，白色）、时间、蓝牙/Wi-Fi连接图标。进入导航态时，此处变为**逐向导航指令**（图标+距离/路名，蓝色）。
    - **RB（右下）**：次级信息与状态区。常驻显示电量、录制状态。覆盖态时，此处显示对应内容，如**音乐**（歌名/歌手，紫色）、**组队**（队友状态/通话中，青色）。
    - **RT（右上）**：轻提示条。用于显示低优先级或持续状态，如“前方学校路段”、“组队已连接”。
    - **LT（左上）**：预警专用区。**所有ADAS预警**（BSD、开门杀、前车碰撞）均在此方位对应的角落（LT或RT）进行**红色闪烁提示**，并伴随简短文字（如“左侧盲区”），严禁使用中央大图标。
- [判断·高] 实现**速度分级（S0-S3）信息密度自适应**。例如，当速度从S1（0-30km/h）升至S3（>80km/h），RB区的音乐信息会从完整歌名简化为图标，最终隐藏，仅保留LB区核心导航或速度信息，确保高速下一瞥可读。

**3. 视觉风格与交互反馈**
- [偏好·--] 建立**全彩功能色语义系统**，通过颜色快速识别信息类型：速度白 (#FFFFFF)、导航蓝 (#4A90E2)、预警红 (#FF4757)、组队青 (#1DD1A1)、音乐紫 (#9B59B6)。
- [判断·高] **A/B光学方案模拟**通过全局CSS变量实现。默认方案A为“全彩OLED模拟”。开发者可通过顶部控制面板一键切换至方案B“单绿光波导模拟”，此时界面将应用滤镜，将所有彩色转换为不同明度的绿色，且不产生额外的状态分支。
- [判断·高] Demo**视觉背景**为纯CSS生成的动态渐变（深蓝至黑色）配合缓慢移动的横向光带，模拟隧道或道路运动感。所有高成本视频/粒子效果默认关闭，仅作为隐藏选项供开发者开启，确保主演示稳定。

**4. 演示叙事与交互控制**
- [偏好·--] **自动剧本**模拟一次完整的短途骑行，叙事节奏遵循“平静（主界面）→ 任务启动（语音唤起导航，蓝色覆盖）→ 途中娱乐（音乐播放，紫色覆盖）→ 突发预警（左侧盲区告警，红色闪烁瞬时态）→ 恢复导航”的情绪曲线，总时长控制在90秒内。
- [事实·高] 界面底部集成**剧本时间轴控制器**，支持播放/暂停/拖拽，右侧集成**手动沙盒控制面板**，可实时触发任意状态（如模拟来电、切换导航路线、调整速度档位），用于现场灵活演示与调试。
- [事实·高] 完整的**键盘快捷键协议**已定义（如按`1`切主界面，`2`切导航，`Space`触发/取消预警），并与自动剧本事件流有明确的抢占逻辑（手动键盘操作将暂停并覆盖自动剧本）。

**5. 技术实现策略**
- [判断·高] 采用 **Vanilla JS + 内联CSS/HTML** 的单文件零依赖方案。代码结构将按模块组织在单一HTML文件内：`<style>`区块定义所有样式与CSS变量；`<script>`区块顶部定义**常量配置区**（包含状态枚举、速度阈值、颜色变量、剧本时间点等），然后是状态机核心逻辑、DOM操作函数、事件监听器及剧本驱动器。
- [判断·中] 为避免单次生成代码质量下降，将采用分段生成策略：先定义数据结构与状态机逻辑框架，再生成核心UI的HTML结构，最后填充CSS与交互脚本，最终由人工合并校验为单一文件。

## 自评：最大风险
**最大风险在于“三层状态机”与“7个页面态+优先级切换”在单文件Vanilla JS环境下的实现复杂度与代码可维护性。** 虽然架构清晰，但在零依赖、单文件的严格约束下，所有状态转换、渲染逻辑、动画控制、事件冲突处理都必须用原生JS手工编码并妥善组织。一旦逻辑出现纰漏（如瞬时态退出后未正确恢复之前的覆盖态），在演示现场可能导致状态卡死，破坏演示流畅度与专业性。这需要极其严谨的代码设计和充分的内部测试来规避。
